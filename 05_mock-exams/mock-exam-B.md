## 解答

### 大問 I 解答

#### (1) f(5) の呼び出し過程と戻り値

```
f(5)
├─ f(4)
│   ├─ f(3)
│   │   ├─ f(2)
│   │   │   ├─ f(1) = 1
│   │   │   └─ f(0) = 0  → f(2) = 1
│   │   └─ f(1) = 1      → f(3) = 2
│   └─ f(2) = 1          → f(4) = 3
└─ f(3) = 2              → f(5) = 5
```

**戻り値: 5**

フィボナッチ数列: f(0)=0, f(1)=1, f(2)=1, f(3)=2, f(4)=3, f(5)=5

#### (2) factorial の空欄

- **(A)** `<=`（n が 1 以下のとき基底ケース）
- **(B)** `1`（0! = 1! = 1）
- **(C)** `n-1`（一つ小さい問題に再帰）

```c
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n-1);
}
```

確認: factorial(5) = 5×4×3×2×1 = 120

#### (3) 末尾再帰版

```c
int factorial_tail(int n, int acc) {
    if (n <= 1) return acc;
    return factorial_tail(n-1, n * acc);
}
```

呼び出し過程: `factorial_tail(5,1)` → `factorial_tail(4,5)` → `factorial_tail(3,20)` → `factorial_tail(2,60)` → `factorial_tail(1,120)` → `120`

#### (4) フィボナッチ反復版

```c
int f_iter(int n) {
    int a, b, c, i;
    if (n <= 1) return n;
    a = 0;
    b = 1;
    for (i = 2; i <= n; i++) {
        c = a + b;
        a = b;
        b = c;
    }
    return b;
}
```

### 大問 II 解答

#### (1) 各 printf の出力

- `(1)` → `3 4`（`*p = a[0] = 3`、`*(p+2) = a[2] = 4`）
- `(2)` → `1`（`*(a+3) = a[3] = 1`）
- `(3)` → `x=20`（`double_val` で x が 10×2=20 になる）
- `(4)` → `x=20 y=20`

  **注意**: swap 前の x=20（double_val後）, y=20。swap 後: x=20, y=20 → 実は x=20, y=20 → swap(&x=20, &y=20) → x=20, y=20 のまま（偶然同じ値）

  より正確には: swap後 x=y の元の値=20, y=x の元の値=20 → `x=20 y=20`

- `(5)` → `1`（`p = p+1 → a[1] = 1`）

#### (2) メモリ状態

```
呼び出し前:
  x → [10]  (アドレス 0x100 と仮定)
  &x → 0x100

呼び出し後:
  x → [20]  (double_val が *p = (*p)*2 を実行)
  p は double_val 内のローカルポインタで、関数終了後は消える
```

#### (3) swap の空欄

```c
void swap(int *a, int *b) {
    int tmp = *a;
    *a = *b;    // (A)
    *b = tmp;   // (B)
}
```

- **(A)** `*a = *b;`
- **(B)** `*b = tmp;`

#### (4) 値渡し swap が動作しない理由

C言語は **値渡し (pass by value)** なので、`swap_wrong(a, b)` を呼ぶと `a` と `b` は元の変数のコピーが作られる。関数内でコピーを入れ替えても、**呼び出し元の変数は変化しない**。ポインタを使うことで「変数のアドレスを渡す」ことができ、間接参照 `*` で元の値を変更できる。

#### (5) バグと修正

**バグ**: ポインタ `p` が初期化されていない（未初期化ポインタへの書き込みは未定義動作）。

**修正**: 変数を宣言してそのアドレスを代入する。

```c
int val = 0;
int *p = &val;
*p = 42;
printf("%d\n", *p);
```

または、整数を直接宣言して使う。

### 大問 III 解答

#### (1) プロトタイプ

```c
int sum_array(int a[], int n);
void normalize(int a[], int n, double result[]);
```

#### (2) 実装

```c
int sum_array(int a[], int n) {
    int i, total = 0;
    for (i = 0; i < n; i++) {
        total += a[i];
    }
    return total;
}

void normalize(int a[], int n, double result[]) {
    int i, max_val;
    max_val = a[0];
    for (i = 1; i < n; i++) {
        if (a[i] > max_val) max_val = a[i];
    }
    for (i = 0; i < n; i++) {
        result[i] = (double)a[i] / max_val;
    }
}
```

#### (3) main() の実装

```c
int main() {
    int a[] = {4, 2, 8, 6};
    int n = 4;
    double norm[4];
    int i;

    printf("合計: %d\n", sum_array(a, n));

    normalize(a, n, norm);
    for (i = 0; i < n; i++) {
        printf("norm[%d] = %.4f\n", i, norm[i]);
    }
    return 0;
}
```

出力:
```
合計: 20
norm[0] = 0.5000
norm[1] = 0.2500
norm[2] = 1.0000
norm[3] = 0.7500
```

#### (4) ゼロ除算への対処

**問題**: 最大値が 0 の場合、`(double)a[i] / max_val` でゼロ除算が発生し、未定義動作（または `inf`/`NaN`）になる。

**対処**:

```c
if (max_val == 0) {
    for (i = 0; i < n; i++) result[i] = 0.0;
    return;
}
```

最大値が 0 の場合は全要素を 0.0 にして早期リターンする。
