## トピック別詳細

### L1: 変数・型・printf/scanf

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年 (毎年必出) |
| **前提知識** | なし（L0 が出発点） |
| **解放するトピック** | L2 演算子、L3 条件分岐 |
| **習得目安** | 2日（2h/日）= 4h |

**コア概念:**
- `int`, `double`, `char`, `float` の4型と用途
- 変数宣言・初期化: `int n = 0;`
- `printf` の書式指定子: `%d %f %lf %c %s`
- `scanf` の `&` 必須ルール: `scanf("%d", &n);`
- 整数除算の落とし穴: `3 / 2 == 1` (小数にならない)

**よくあるエラー:**
- `double x = 3 / 2;` → x は 1.0 になる（右辺が整数演算）
- `scanf("%d", n)` → `&n` を忘れて実行時クラッシュ
- `%d` で `double` を出力 → ゴミ値が出る

### L2: 演算子・式・型変換

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年 |
| **前提知識** | L1 変数・型 |
| **解放するトピック** | L3 条件分岐 |
| **習得目安** | 2日 = 4h |

**コア概念:**
- 算術演算子: `+  -  *  /  %`
- 比較演算子: `==  !=  <  >  <=  >=`
- 論理演算子: `&&  ||  !`
- インクリメント前置 `++n` vs 後置 `n++`（式の中での違い）
- キャスト: `(double)n / m` で実数除算
- 優先順位: `*` > `+` > `==` > `&&` > `||`

**よくあるエラー:**
- `n = n++` など副作用の二重評価（未定義動作）
- `x = 5 / 2 * 2.0` → 5/2 が先に整数計算されて 4.0 になる
- `a == b == c` を「a と b と c が等しい」と誤解（実際は `(a==b)==c`）

### L3: 条件分岐

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年 |
| **前提知識** | L1, L2 |
| **解放するトピック** | L4 ループ |
| **習得目安** | 2日 = 4h |

**コア概念:**
- `if / else if / else` の連鎖
- ネストした `if`（インデント追跡が重要）
- `switch / case / break / default`
- `break` なしフォールスルーの動作
- 三項演算子: `x > 0 ? x : -x`

**よくあるエラー:**
- `switch` の `break` 忘れ → 次の `case` も実行される
- `if (x = 5)` (`==` を `=` と書く) → 常に真になる
- ネスト `if` の `else` がどの `if` に対応するか混乱

### L4: ループ

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年（ループトレースは毎年出題） |
| **前提知識** | L1, L2, L3 |
| **解放するトピック** | L5 配列、L7 関数、L9 ポインタ |
| **習得目安** | 3日 = 6h |

**コア概念:**
- `for(init; cond; update)` の3部構成
- `while(cond)` と `do-while(cond)` の違い（do-while は必ず1回実行）
- `break`（ループ脱出）と `continue`（次の反復へ）
- ネストループの外側・内側変数の追跡
- ループ不変条件の考え方

**よくあるエラー:**
- `for(int i=0; i<n; i++)` vs `i<=n` の境界 off-by-one エラー
- ネストループで `break` が内側だけを抜ける（外側には効かない）
- 無限ループ: 更新式を忘れた `while(n > 0) { printf... }`

### L5: 配列

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年 |
| **前提知識** | L4 ループ |
| **解放するトピック** | L6 文字列、L10 ソート・探索 |
| **習得目安** | 3日 = 6h |

**コア概念:**
- 宣言: `int a[10];`、初期化: `int a[] = {1,2,3};`
- インデックスは 0 〜 n-1（最後は `a[n-1]`）
- ループで全要素走査: `for(int i=0; i<n; i++)`
- 2次元配列: `int mat[3][4];`、`mat[row][col]`
- 配列を関数に渡すと先頭アドレスが渡る（値コピーではない）

**よくあるエラー:**
- `a[n]` アクセス（範囲外）→ 未定義動作
- `int a[5] = {1,2,3,4,5,6};` → コンパイルエラー（初期化子が多い）
- 2次元配列の行/列のインデックスを逆に書く

### L6: 文字列（char配列）

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 |
| **試験頻度** | 2/3 年 |
| **前提知識** | L5 配列 |
| **解放するトピック** | L9 ポインタ（文字列ポインタ） |
| **習得目安** | 2日 = 4h |

**コア概念:**
- 文字列 = `char` 配列 + ヌル終端 `\0`
- 宣言: `char s[20] = "hello";`（サイズは文字数+1）
- `strlen(s)` → 文字数（`\0` を含まない）
- `strcpy(dst, src)` → 文字列コピー
- `strcmp(s1, s2)` → 0なら等しい、負なら s1<s2
- 文字列は `==` で比較できない → `strcmp` 必須

**よくあるエラー:**
- `char s[5] = "hello";` → ヌル終端用スペースがない（バッファオーバーフロー）
- `if (s == "hello")` → ポインタ比較になって常に偽
- `strlen` の戻り値は `size_t`（符号なし）→ `int` と比較で警告

### L7: 関数

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 ★ |
| **試験頻度** | 3/3 年 |
| **前提知識** | L4 ループ、L5 配列 |
| **解放するトピック** | L8 再帰、L9 ポインタ引数 |
| **習得目安** | 3日 = 6h |

**コア概念:**
- 関数定義: `戻り値型 関数名(引数) { ... return 値; }`
- `void` 関数: 戻り値なし
- 値渡し（コピー）vs アドレス渡し（ポインタ）の違い
- スコープ: ローカル変数はブロック内のみ有効
- プロトタイプ宣言（関数を使う前に宣言する慣例）

**よくあるエラー:**
- 値渡しで引数を変えても呼び元に反映されない（`swap` 問題）
- `return` を忘れると未定義の戻り値
- ローカル変数のアドレスを return する（dangling pointer）

### L8: 再帰関数

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 |
| **試験頻度** | 2/3 年 |
| **前提知識** | L7 関数 |
| **解放するトピック** | L10 高度なアルゴリズム |
| **習得目安** | 3日 = 6h |

**コア概念:**
- 基底ケース（base case）: 再帰を止める条件
- 再帰ケース（recursive case）: 自分自身を呼ぶ
- 階乗: `n! = n * (n-1)!`、`f(0)=1`
- フィボナッチ: `fib(n) = fib(n-1) + fib(n-2)`、`fib(0)=fib(1)=1`
- スタックの展開をツリー図で追う技法
- 末尾再帰 vs 非末尾再帰

**よくあるエラー:**
- 基底ケースが ≤1 ではなく `==0` だけ → 負の引数で無限再帰
- `fib(4)` のトレースを暗算しようとして間違える（必ず木で展開する）
- スタックオーバーフロー: 終了条件のない再帰

### L9: ポインタ

| 項目 | 内容 |
|---|---|
| **Tier** | Tier 1 |
| **試験頻度** | 2/3 年 |
| **前提知識** | L5 配列、L7 関数 |
| **解放するトピック** | L6 文字列ポインタ、L10 動的配列 |
| **習得目安** | 4日 = 8h |

**コア概念:**
- `int *p;` はポインタ変数（`int` のアドレスを格納）
- `p = &a;` で `a` のアドレスを取得
- `*p` で間接参照（ポインタが指す値にアクセス）
- ポインタ算術: `p+1` は次の `int` のアドレス（+4バイト）
- 配列と等価: `a[i] == *(a+i)`
- 関数引数での `swap`: `void swap(int *a, int *b) { int t=*a; *a=*b; *b=t; }`

**よくあるエラー:**
- `*p` と `p` の混同（値 vs アドレス）
- 未初期化ポインタの間接参照（セグメンテーション違反）
- `p++` と `(*p)++` の違い
