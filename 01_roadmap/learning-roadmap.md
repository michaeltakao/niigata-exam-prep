以下の問題を解いてください。正しく答えられるレベルは**スキップ可能**です。

| # | 問題 | 正解 | 対応レベル |
|---|---|---|---|
| 1 | `int x = 7; printf("%d", x);` の出力は？ | `7` | L1 スキップ |
| 2 | `int a=5, b=2; printf("%d %d", a/b, a%b);` の出力は？ | `2 1` | L2 スキップ |
| 3 | `int x=3; if(x>5) printf("A"); else printf("B");` の出力は？ | `B` | L3 スキップ |

## L1: 変数・型・printf/scanf

### エントリー条件
- L0 完了（プログラムの基本構造を理解）

### 修了条件（3つ全てができること）
1. `int`, `double`, `char` の変数を宣言・初期化して正しい書式で printf できる
2. `scanf` で整数・実数を入力し変数に格納できる
3. 整数除算と実数除算の違いを言語化できる

### コア概念
- **データ型 4種**: `int`（整数）、`double`（実数）、`char`（文字）、`float`（単精度実数）
- **変数宣言**: `型 変数名 = 初期値;`
- **printf 書式**: `%d`=int, `%f`=float/double, `%lf`=double(scanfのみ), `%c`=char, `%s`=文字列
- **scanf の & 必須**: `scanf("%d", &n);`（配列・ポインタは & 不要）
- **整数除算**: `int / int` は小数部が切り捨て → `3/2 == 1`
- **型変換（キャスト）**: `(double)3 / 2` → `1.5`

### 典型エラー
- `double x = 3 / 2;` → `x == 1.0`（右辺が整数計算後に double に変換）
- `scanf("%d", n)` → & を忘れて実行時クラッシュ（Tier1 失点パターン）

### 自己テスト
1. `int a=10, b=3; printf("%d", a/b);` → **3**
2. `double x = (double)10/3; printf("%.1f", x);` → **3.3**
3. `char c = 'A'; printf("%c %d", c, c);` → **A 65**

2日（4h）

## L2: 演算子・式・型変換

### エントリー条件
- L1 完了

### 修了条件
1. 演算子優先順位表を見ずに `a + b * c == d && e` を正しく括弧で展開できる
2. `n++` と `++n` の差を含む式の評価値を答えられる
3. 型変換（暗黙・明示）が起きる場面を3パターン挙げられる

### コア概念
- **算術**: `+  -  *  /  %`（% は剰余、整数のみ）
- **比較**: `==  !=  <  >  <=  >=`（戻り値は 0 or 1）
- **論理**: `&&`（AND）、`||`（OR）、`!`（NOT）
- **代入**: `=  +=  -=  *=  /=  %=`
- **前置・後置**: `int n=5; int a=n++;` → `a==5, n==6` / `int b=++n;` → `b==7, n==7`
- **優先順位（高→低）**: `!` > `* / %` > `+ -` > `< > <= >=` > `== !=` > `&&` > `||` > `=`
- **短絡評価**: `&&` は左が偽なら右を評価しない

### 典型エラー
- `a == b == c` を「3者等しい」と誤解（実際は `(a==b)==c`）
- `5 / 2 * 2.0` → `2 * 2.0 = 4.0`（`/` が先に整数計算）

### 自己テスト
1. `int n=3; printf("%d %d", n++, n);` → **3 4**（後置なので式中では 3）
2. `int x = 10 % 3;` → **1**
3. `int a=1, b=0; printf("%d", a || b && !a);` → **1**（`!a==0`、`b&&0==0`、`1||0==1`）

### 所要時間
2日（4h）

## L3: 条件分岐

### エントリー条件
- L1, L2 完了

### 修了条件
1. ネストした `if-else if-else` 連鎖を読んで正しい分岐経路をトレースできる
2. `switch` で `break` なしフォールスルーの実行結果を答えられる
3. 三項演算子 `? :` を読んで評価できる

### コア概念
- **if-else 連鎖**: 先に真になった条件だけ実行、残りはスキップ
- **ネスト**: `else` は直前の `if` に対応する（インデントで視認）
- **switch-case**: 定数式のみ。`break` なしで次 case に fallthrough
- **三項演算子**: `条件 ? 真の値 : 偽の値`

### 典型エラー
- `if (x = 5)` → 代入が起きて常に真（`==` を書くこと）
- `switch` の `break` 忘れによる意図しない fallthrough

### 自己テスト
```c
int x = 3;
if (x > 5)      printf("A");
else if (x > 1) printf("B");
else            printf("C");
```
1. 上の出力は? → **B**
2. `int x=2; switch(x){case 1:printf("one"); case 2:printf("two"); case 3:printf("three");}` → **twothree**（break なし fallthrough）
3. `int a = (4 > 3) ? 10 : 20;` → **a == 10**

### 所要時間
2日（4h）

## L4: ループ

### エントリー条件
- L1, L2, L3 完了

### 修了条件
1. `for` / `while` / `do-while` のトレース表（変数の値の変化）を正しく埋められる
2. `break` と `continue` を含むネストループの動作を追える
3. 典型パターン（累積和・積、逆順出力、二重ループ九九）を自力で書ける

### コア概念
- **for**: `for(init; cond; update)` — 回数が決まっている場合
- **while**: `while(cond)` — 条件が複雑な場合
- **do-while**: 最低1回実行してから条件チェック
- **break**: ループを即脱出（switch 内も同様）
- **continue**: 残り処理をスキップして次の反復へ
- **ネストループ**: 外側 i・内側 j で 2D 操作

### 典型エラー
- `for(int i=0; i<=n; i++)` → n+1 回実行（off-by-one）
- `for` の更新式を忘れた `while` → 無限ループ
- ネストの `break` が内側だけ抜ける（外側には効かない）

### 自己テスト
```c
int s = 0;
for (int i = 1; i <= 5; i++) {
    if (i % 2 == 0) continue;
    s += i;
}
printf("%d", s);
```
1. 出力は? → **9**（1+3+5）

```c
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break;
        printf("%d%d ", i, j);
    }
}
```
2. 出力は? → **00 10 20**

3. 1から100の整数の和を while で書け:  
   → `int s=0, i=1; while(i<=100){s+=i; i++;} printf("%d",s);`

3日（6h）

- R7: while ループを使った探索

## L5: 配列

### エントリー条件
- L4 完了

### 修了条件
1. 配列の宣言・初期化・全要素走査をコードで書ける
2. 2次元配列の `a[row][col]` アクセスパターンを正しく使える
3. 配列を関数に渡したとき何が渡るか（アドレス）を説明できる

### コア概念
- **1次元**: `int a[5];` → `a[0]` 〜 `a[4]`
- **初期化**: `int a[] = {1,2,3};` → サイズ自動
- **走査**: `for(int i=0; i<n; i++)` が基本パターン
- **2次元**: `int m[3][4];` → `m[0][0]` 〜 `m[2][3]`
- **配列と関数**: `void f(int a[], int n)` — 先頭アドレスが渡る（値コピーではない）

### 典型エラー
- `a[5]` アクセス（サイズ5の配列の6番目）→ 未定義動作
- `sizeof(a)/sizeof(a[0])` が関数内で使えない（先頭アドレスになるため）

### 自己テスト
```c
int a[] = {5, 3, 8, 1, 4};
int max = a[0];
for (int i=1; i<5; i++) {
    if (a[i] > max) max = a[i];
}
printf("%d", max);
```
1. 出力は? → **8**
2. `int a[3][3] = {{1,2,3},{4,5,6},{7,8,9}}; printf("%d", a[1][2]);` → **6**
3. `void double_arr(int a[], int n)` で全要素を2倍にするコードを書け:  
   → `for(int i=0;i<n;i++) a[i]*=2;`

## L6: 関数

### エントリー条件
- L4, L5 完了

### 修了条件
1. 戻り値あり・なし両方の関数を定義・呼び出しできる
2. ローカル変数のスコープを正しく説明でき、穴埋め問題を解ける
3. 再帰関数の基底ケース・再帰ケースを特定してトレースできる

### コア概念
- **定義**: `戻り値型 名前(型 引数1, 型 引数2) { ... return 値; }`
- **void 関数**: `void print_hello(void) { printf("hello\n"); }`
- **値渡し**: 引数の値がコピーされる → 関数内で変更しても呼び元に影響しない
- **スコープ**: ローカル変数は `{}` ブロック内のみ有効
- **再帰**: 自分自身を呼ぶ関数 — 基底ケースが必須
- **階乗**: `int fact(int n){ if(n<=1) return 1; return n*fact(n-1); }`

### 典型エラー
- `int swap(int a, int b)` では呼び元の変数が変わらない
- 再帰の基底ケース欠落 → スタックオーバーフロー

### 自己テスト
```c
int add(int a, int b) { return a + b; }
int mul(int a, int b) { return a * b; }
printf("%d", add(mul(2,3), mul(4,5)));
```
1. 出力は? → **26**（`add(6,20)`）
2. `fact(5)` は? → **120**
3. `fib(5)` (`fib(n)=fib(n-1)+fib(n-2)`, `fib(0)=fib(1)=1`) は? → **8**

## L7: ポインタ

### エントリー条件
- L5 配列、L6 関数 完了

### 修了条件
1. `*` と `&` の意味をそれぞれ説明し、コード例で示せる
2. ポインタ算術 `p+1`、`*(p+i)` が配列と等価であることを示せる
3. ポインタ引数で `swap` 関数を実装できる

### コア概念
- **ポインタ変数**: `int *p;` — int のアドレスを格納する変数
- **アドレス取得**: `p = &a;`
- **間接参照**: `*p` — ポインタが指す場所の値
- **配列との等価性**: `a[i] == *(a+i)`
- **ポインタ引数での swap**:
  ```c
  void swap(int *a, int *b) {
      int tmp = *a; *a = *b; *b = tmp;
  }
  swap(&x, &y);
  ```
- **文字列ポインタ**: `char *s = "hello";`

### 典型エラー
- `*p` と `p` の混同（値 vs アドレス）
- 未初期化ポインタの `*` 参照（即クラッシュ）
- `p++` はポインタ自体を移動、`(*p)++` はポイント先の値を増加

### 自己テスト
```c
int a[] = {10, 20, 30, 40, 50};
int *p = a + 1;
printf("%d %d %d", *p, *(p+2), p[-1]);
```
1. 出力は? → **20 40 10**
2. `swap(&x, &y)` 後 `printf("%d %d", x, y)` — x=3, y=7 で呼んだら? → **7 3**
3. `char *s="hello"; printf("%c", *(s+3));` → **l**

### 所要時間
4日（8h）
