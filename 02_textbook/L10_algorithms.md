# L10 アルゴリズム — ソートと探索（最頻出パターン）

> **難易度**: ★★★☆☆ | **前提**: L05 配列、L07 関数 | **試験重要度**: Tier 1 ★（全年出題）

---

## 1. コンセプト概要

**アルゴリズム**とは「問題を解くための手順の集まり」です。プログラミングでは、データの整列（ソート）と検索（探索）が最も基本的なアルゴリズムです。

この章では試験で繰り返し出題される4つのアルゴリズムを学びます:

| アルゴリズム | 目的 | 試験頻度 |
|---|---|---|
| **バブルソート** | 配列を昇順/降順に並べ替え | ★★★（最頻出） |
| **選択ソート** | 配列を並べ替え | ★★☆ |
| **線形探索** | 配列から値を検索 | ★★★ |
| **二分探索** | ソート済み配列から高速検索 | ★★☆ |

---

## 2. なぜ必要か

日常生活でもソートと探索は欠かせません:

- **ソート**: 成績を高い順に並べる、出席番号順に並べる
- **探索**: 名簿から特定の人を探す、辞書で単語を引く

プログラムも同様で、データを整理してから効率よく検索するのが基本パターンです。

新潟大学の試験では「バブルソートの空欄を埋めよ」「このプログラムの出力を答えよ」という問題が R6〜R8 の全年度で出題されています。

## 3. 現実世界のアナロジー

### バブルソートのアナロジー: トランプのカード並べ替え

5枚のカード `3, 1, 4, 1, 5` を昇順に並べ替えると想像してください。

あなたは左から順に隣どうしを比べて、左が大きければ交換していきます:
- `3, 1` → 3 > 1 なので交換 → `1, 3, 4, 1, 5`
- `3, 4` → 3 < 4 なので交換しない
- `4, 1` → 4 > 1 なので交換 → `1, 3, 1, 4, 5`
- `4, 5` → 4 < 5 なので交換しない

1回全体を走査すると、最大値（5）が右端に「浮かび上がった」ように移動します。これが **バブル（泡）ソート**の名前の由来です。

### 線形探索のアナロジー: 出席簿を1ページずつめくる

「田中さん」を探すとき、名簿を最初から1人ずつ確認していきます。見つかったらその位置を返す、最後まで見つからなければ「いない」と報告します。

### 二分探索のアナロジー: 辞書で単語を調べる

辞書で「moon」を探すとき、真ん中のページを開きます。そのページの単語が「moon」より前なら後半を、後なら前半を開きます。これを繰り返すと効率よく見つかります。

## 4. 構文説明

### バブルソート

```c
#include <stdio.h>

void bubble_sort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {        // n-1 パス繰り返す
        for (int j = 0; j < n - 1 - i; j++) { // 各パスで比較する範囲
            if (a[j] > a[j + 1]) {            // 左が右より大きければ
                int tmp = a[j];               // ① tmp に左を保存
                a[j] = a[j + 1];              // ② 左に右を代入
                a[j + 1] = tmp;              // ③ 右に tmp（元の左）を代入
            }
        }
    }
}

int main(void) {
    int a[] = {3, 1, 4, 1, 5};
    int n = 5;
    bubble_sort(a, n);
    for (int i = 0; i < n; i++) {
        printf("%d ", a[i]);
    }
    printf("\n");
    return 0;
}
```

**重要なポイント:**

- 外側のループ `i`: 何パス目かを表す（0〜n-2）
- 内側のループ `j`: 比較する位置（0〜n-2-i）
- `n-1-i` が重要: i パス目終了後、右端 i 個はすでに確定しているので比較不要
- 3ステップのスワップ (`tmp`, `a[j]=a[j+1]`, `a[j+1]=tmp`) は試験で穴埋め問題になる

### 選択ソート

```c
void selection_sort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min_idx = i;                      // 最小値のインデックス
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[min_idx]) {
                min_idx = j;                  // より小さい値を発見
            }
        }
        if (min_idx != i) {                   // 最小値が i でなければ交換
            int tmp = a[i];
            a[i] = a[min_idx];
            a[min_idx] = tmp;
        }
    }
}
```

### 線形探索

```c
int linear_search(int a[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (a[i] == key) {
            return i;   // 見つかったインデックスを返す
        }
    }
    return -1;          // 見つからなかった場合
}
```

### 二分探索（ソート済み配列が必要）

```c
int binary_search(int a[], int n, int key) {
    int left = 0, right = n - 1;
    while (left <= right) {              // left > right になったら終了
        int mid = (left + right) / 2;   // 中央インデックス
        if (a[mid] == key) {
            return mid;                  // 発見
        } else if (a[mid] < key) {
            left = mid + 1;             // 右半分を探す
        } else {
            right = mid - 1;            // 左半分を探す
        }
    }
    return -1;                          // 見つからなかった
}
```

## 5. ステップ実行トレース

### バブルソート: 配列 {5, 2, 8, 1, 9} の全トレース

**i=0 パス（全5要素を比較）:**

| j | a[j] | a[j+1] | swap? | 配列状態 |
|---|---|---|---|---|
| 0 | 5 | 2 | YES (5>2) | {**2, 5**, 8, 1, 9} |
| 1 | 5 | 8 | NO  (5<8) | {2, 5, 8, 1, 9} |
| 2 | 8 | 1 | YES (8>1) | {2, 5, **1, 8**, 9} |
| 3 | 8 | 9 | NO  (8<9) | {2, 5, 1, 8, 9} |

→ i=0 終了: **最大値 9 が末尾に確定**

**i=1 パス（末尾1個除く）:**

| j | a[j] | a[j+1] | swap? | 配列状態 |
|---|---|---|---|---|
| 0 | 2 | 5 | NO  (2<5) | {2, 5, 1, 8, 9} |
| 1 | 5 | 1 | YES (5>1) | {2, **1, 5**, 8, 9} |
| 2 | 5 | 8 | NO  (5<8) | {2, 1, 5, 8, 9} |

→ i=1 終了: **8 が確定**

**i=2 パス:**

| j | a[j] | a[j+1] | swap? | 配列状態 |
|---|---|---|---|---|
| 0 | 2 | 1 | YES (2>1) | {**1, 2**, 5, 8, 9} |
| 1 | 2 | 5 | NO  (2<5) | {1, 2, 5, 8, 9} |

→ i=2 終了: **5 が確定**

**i=3 パス:**

| j | a[j] | a[j+1] | swap? | 配列状態 |
|---|---|---|---|---|
| 0 | 1 | 2 | NO  (1<2) | {1, 2, 5, 8, 9} |

→ i=3 終了: **2 が確定**

**最終結果: {1, 2, 5, 8, 9}** ✓

### 二分探索: 配列 {1, 3, 5, 7, 9, 11, 13} で 7 を探す

```
配列インデックス:  0  1  2  3  4   5   6
配列の値:         [1, 3, 5, 7, 9, 11, 13]

Step 1: left=0, right=6
        mid=(0+6)/2=3  →  a[3]=7  →  key と一致! → return 3
```

→ 1回の比較で発見（最良ケース）。

## 7. よくある間違い

### ① 内側ループの上限を間違える

```c
// WRONG: j < n-1 (最後のパスも末尾まで比較してしまう)
for (int j = 0; j < n - 1; j++) {

// RIGHT: j < n-1-i (確定済みの末尾 i 個を除く)
for (int j = 0; j < n - 1 - i; j++) {
```

`n-1-i` でなくても動作はするが、無駄な比較が発生する。試験では `n-1-i` が正解。

### ② スワップで tmp を使わない

```c
// WRONG: a[j] の値が失われる!
a[j] = a[j + 1];
a[j + 1] = a[j];  // ← もう a[j] は上書きされているので同じ値になる

// RIGHT: 必ず tmp で保存
int tmp = a[j];
a[j] = a[j + 1];
a[j + 1] = tmp;
```

### ③ 二分探索でソートされていない配列を使う

```c
int a[] = {5, 1, 3, 7, 2};   // ソートされていない!
// binary_search(a, 5, 3) は誤った結果を返す可能性がある
// 必ず先にソートしてから使うこと
```

### ④ 二分探索の終了条件を間違える

```c
// WRONG: left < right (left==right の時に見つかる1要素を見逃す)
while (left < right) {

// RIGHT:
while (left <= right) {
```

### ⑤ 線形探索の戻り値でインデックスをそのまま使う

```c
int result = linear_search(a, n, 42);
printf("%d\n", a[result]);  // WRONG: result が -1 の場合にクラッシュ!

// RIGHT: 必ず -1 チェックをする
if (result == -1) {
    printf("見つからない\n");
} else {
    printf("index=%d, 値=%d\n", result, a[result]);
}
```

## 8. ミニクイズ

**Q1.** バブルソートで `{4, 2, 7, 1}` を昇順ソートするとき、i=0 の最初の比較（j=0）でスワップは発生するか？

<details><summary>答え</summary>
YES。a[0]=4 > a[1]=2 なのでスワップ。配列は `{2, 4, 7, 1}` になる。
</details>

**Q2.** バブルソートの内側ループで `j < n-1-i` の代わりに `j < n-1` と書いた場合、プログラムは正しく動くか？

<details><summary>答え</summary>
動く（正しい結果が出る）が、無駄な比較が増えて非効率。試験では `n-1-i` が正解として求められる。
</details>

**Q3.** 線形探索で値が見つからなかった場合、何を返すのが慣例か？

<details><summary>答え</summary>
`-1` を返す。インデックスは 0 以上なので、-1 は「見つからなかった」という意味として使える。
</details>

**Q4.** 二分探索を使う前提条件は何か？

<details><summary>答え</summary>
配列がソート（整列）済みであること。ソートされていない配列に使うと誤った結果になる。
</details>

**Q5.** 配列 `{1, 3, 5, 7, 9}` に対して binary_search(a, 5, 6) を実行すると何が返るか？

<details><summary>答え</summary>
`-1`（6は配列に存在しないため）。
トレース: left=0,right=4,mid=2→a[2]=5<6→left=3。left=3,right=4,mid=3→a[3]=7>6→right=2。left>right → return -1。
</details>

## 9. 例題

### 例題 1: バブルソートの完全実装とトレース

**問題:** 配列 `{3, 1, 4, 1, 5}` をバブルソートで昇順に並べ替えよ。各パスの結果を示せ。

```c
#include <stdio.h>

void bubble_sort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                int tmp = a[j];
                a[j] = a[j + 1];
                a[j + 1] = tmp;
            }
        }
    }
}

int main(void) {
    int a[] = {3, 1, 4, 1, 5};
    int n = 5;
    bubble_sort(a, n);
    for (int i = 0; i < n; i++) {
        printf("%d ", a[i]);
    }
    printf("\n");
    return 0;
}
```

**各パスの結果:**
```
初期: {3, 1, 4, 1, 5}
i=0:  {1, 3, 1, 4, 5}  ← 5 確定
i=1:  {1, 1, 3, 4, 5}  ← 4 確定
i=2:  {1, 1, 3, 4, 5}  ← 3 確定（変化なし）
i=3:  {1, 1, 3, 4, 5}  ← 完了
出力: 1 1 3 4 5
```

### 例題 2: 線形探索（発見・未発見両方の処理）

**問題:** 配列から値を検索し、見つかれば "found at index X"、なければ "not found" を表示せよ。

```c
#include <stdio.h>

int linear_search(int a[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (a[i] == key) return i;
    }
    return -1;
}

int main(void) {
    int a[] = {10, 30, 20, 50, 40};
    int n = 5;

    int idx = linear_search(a, n, 20);
    if (idx == -1) {
        printf("not found\n");
    } else {
        printf("found at index %d\n", idx);
    }

    idx = linear_search(a, n, 99);
    if (idx == -1) {
        printf("not found\n");
    } else {
        printf("found at index %d\n", idx);
    }
    return 0;
}
```

**出力:**
```
found at index 2
not found
```

### 例題 3: 二分探索のトレース

**問題:** ソート済み配列 `{1, 3, 5, 7, 9, 11}` から 7 を二分探索で見つけよ。

```c
#include <stdio.h>

int binary_search(int a[], int n, int key) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = (left + right) / 2;
        if (a[mid] == key)      return mid;
        else if (a[mid] < key)  left = mid + 1;
        else                    right = mid - 1;
    }
    return -1;
}

int main(void) {
    int a[] = {1, 3, 5, 7, 9, 11};
    int n = 6;
    int result = binary_search(a, n, 7);
    if (result != -1) {
        printf("found at index %d\n", result);
    } else {
        printf("not found\n");
    }
    return 0;
}
```

**トレース:**
```
配列: [1, 3, 5, 7, 9, 11]  key=7
       0  1  2  3  4   5

Step 1: left=0, right=5, mid=2
        a[2]=5 < 7 → left=3

Step 2: left=3, right=5, mid=4
        a[4]=9 > 7 → right=3

Step 3: left=3, right=3, mid=3
        a[3]=7 == 7 → return 3

出力: found at index 3
```

## 10. 新潟大学試験との関連

**出題実績:** ソート・探索問題は R6・R7・R8 の**全年度で出題**されており、最頻出トピックです。

**典型的な出題パターン:**

1. **バブルソートの空欄埋め（最頻出）:**
   ```c
   if (a[j] (A) a[j+1]) {   // (A) を埋めよ: 答え >
       int tmp = a[j];
       a[j] = (B);            // (B) を埋めよ: 答え a[j+1]
       a[j+1] = (C);          // (C) を埋めよ: 答え tmp
   }
   ```

2. **実行結果を答えよ:** バブルソートを途中まで実行した配列状態を問う

3. **内側ループの条件を問う:** `j < n-1-i` の `n-1-i` 部分の穴埋め

4. **線形探索の実装:** 関数を自分で書かせる

**試験対策のポイント:**
- バブルソートの**スワップ3行**（tmp→a[j]=a[j+1]→a[j+1]=tmp）は暗記すること
- ループ条件 `j < n-1-i` の `i` の役割を説明できるようにすること
- 実行トレース問題では**変数トレース表**を紙に書いて解くこと
