# L08 ポインタ — メモリアドレスを扱う（最難関トピック）

> **警告:** ポインタはC言語で最も難しいトピックです。  
> 焦らず、メモリ図を紙に描きながら読み進めてください。

---

## 1. コンセプト概要

**ポインタ（pointer）** とは「変数のアドレス（メモリ上の住所）を保存する変数」です。

普通の変数が「値」を保存するのに対して、ポインタは「どこに値があるか（住所）」を保存します。

```
普通の変数:  int x = 5;    →  x の中身 = 5（値）
ポインタ変数: int *p = &x; →  p の中身 = x のアドレス（住所）
```

ポインタを通じて、「別の変数の値を読んだり書いたり」できます。

---

## 2. なぜ必要か

**問題1: 関数から複数の値を「返したい」**

```c
// 値渡しでは x も y も変わらない（L07参照）
void swap_wrong(int a, int b) {
    int tmp = a;
    a = b;
    b = tmp;
    // 呼び出し元には何も伝わらない！
}
```

ポインタを使うと、呼び出し元の変数を直接操作できます。

**問題2: 大きなデータをコピーせずに渡したい**

10万要素の配列を「値コピー」で渡すと遅い。ポインタなら「住所」だけ渡せばよい。

**問題3: 動的なデータ構造（リスト・木）を作りたい**

ポインタなしには、高度なデータ構造は実装できません。

## 3. 現実世界のアナロジー

**ポインタ ≒ 地図上の住所**

- 「田中さんの家」という概念 → 変数 `x`
- 「田中さんの家の住所（〒950-2181 新潟市...）」 → `&x`（アドレス）
- 「この住所を書いたメモ」 → ポインタ変数 `p`
- 「住所のメモを持って実際に家に行く」 → `*p`（デリファレンス）

```
メモ (p) には: 〒950-2181 が書いてある
その住所に行くと: 田中さん (x = 5) がいる
*p は「住所を手がかりに実際の人を見つける」操作
```

**別のアナロジー: 図書館の本棚**

- 本そのもの → 変数の値
- 本棚の番号（302番棚）→ アドレス
- 図書カード（「302番棚にある」と書いてある）→ ポインタ
- 図書カードを見て実際の棚に行く → デリファレンス（`*p`）

## 4. 構文説明

### 基本の3操作

**操作1: アドレスを取得する (`&`)**
```c
int x = 5;
&x   // x が格納されているメモリアドレスを返す
     // 例えば 0x7fff5fbff58c のような16進数値
```

**操作2: ポインタ変数を宣言する (`*` in 宣言)**
```c
int *p;     // 「int型の値を指すポインタ」変数 p を宣言
            // ← この * は「宣言の * 」= 「ポインタですよ」の印
```

**操作3: ポインタ変数に代入してデリファレンスする**
```c
p = &x;     // p に x のアドレスを代入（p は x を「指している」）
*p          // p の指す先の値を読む = x の値 = 5
            // ← この * は「デリファレンスの * 」= 「住所に行って値を取る」操作
```

### 宣言の * とデリファレンスの * は別物！

```c
int *p = &x;   // ← 宣言の * : "p はポインタ変数"
*p = 100;      // ← デリファレンスの * : "p が指す場所に 100 を書く"
```

同じ `*` 記号ですが、**意味が全く異なります**。慣れるまで意識して区別してください。

### ポインタ算術

```c
int a[] = {10, 20, 30, 40};
int *p = a;    // a は &a[0] と同じ（配列の先頭アドレス）

p + 0          // a[0] のアドレス → *( p+0) = 10
p + 1          // a[1] のアドレス → *(p+1) = 20  (4バイト先)
p + 2          // a[2] のアドレス → *(p+2) = 30  (8バイト先)

// a[i] と *(a+i) は等価！
a[2]    ==    *(a + 2)    ==    *(p + 2)    // すべて 30
```

## 5. ステップ実行トレース

次のプログラムを4ステップで追います:

```c
#include <stdio.h>
int main() {
    int x = 5;
    int y = 10;
    int *p = &x;
    
    printf("x=%d *p=%d\n", x, *p);   // Step 1
    *p = 20;
    printf("x=%d *p=%d\n", x, *p);   // Step 2
    p = &y;
    printf("y=%d *p=%d\n", y, *p);   // Step 3
    (*p)++;
    printf("y=%d *p=%d\n", y, *p);   // Step 4
    return 0;
}
```

---

**初期状態（int x=5; int y=10; int *p=&x; の直後）:**

```
アドレス:  1000    1004    1008
          ┌────┐  ┌────┐  ┌────┐
内容:     │  5 │  │ 10 │  │1000│
          └────┘  └────┘  └────┘
変数:      x       y       p

p は 1000 を指している（= x のアドレスが 1000 と仮定）
```

---

**Step 1: `printf("x=%d *p=%d\n", x, *p);`**

```
x = 5 (直接読む)
*p → アドレス 1000 に行って値を読む → 5

出力: x=5 *p=5
```

---

**Step 2: `*p = 20;` の後 `printf(...)`**

```
*p = 20 → アドレス 1000 (= x の場所) に 20 を書く

アドレス:  1000    1004    1008
          ┌────┐  ┌────┐  ┌────┐
内容:     │ 20 │  │ 10 │  │1000│
          └────┘  └────┘  └────┘
変数:      x       y       p
           ↑
        x が 20 に変わった！

出力: x=20 *p=20
```

---

**Step 3: `p = &y;` の後 `printf(...)`**

```
p = &y → p に y のアドレス（1004）を代入
         p が指す先が y に切り替わる

アドレス:  1000    1004    1008
          ┌────┐  ┌────┐  ┌────┐
内容:     │ 20 │  │ 10 │  │1004│
          └────┘  └────┘  └────┘
変数:      x       y       p
                   ↑ ←← p が指している

y = 10、*p = *(アドレス1004) = 10

出力: y=10 *p=10
```

---

**Step 4: `(*p)++;` の後 `printf(...)`**

```
(*p)++ → *p の値（=y の値 = 10）を 1 増やす → y = 11

アドレス:  1000    1004    1008
          ┌────┐  ┌────┐  ┌────┐
内容:     │ 20 │  │ 11 │  │1004│
          └────┘  └────┘  └────┘
変数:      x       y       p
                   ↑ y が 11 に変わった

出力: y=11 *p=11
```

**最終出力まとめ:**
```
x=5 *p=5
x=20 *p=20
y=10 *p=10
y=11 *p=11
```

## 6. 視覚的メモリ図

### ポインタと配列の関係

```c
int a[] = {10, 20, 30};
int *p = a;
```

```
メモリ:
アドレス: 2000    2004    2008
         ┌────┐  ┌────┐  ┌────┐
内容:    │ 10 │  │ 20 │  │ 30 │
         └────┘  └────┘  └────┘
          a[0]    a[1]    a[2]
           ↑
        p = 2000

p+0 → 2000 → *( p+0) = 10 = a[0]
p+1 → 2004 → *(p+1) = 20 = a[1]  ← int は 4 バイトなので +4
p+2 → 2008 → *(p+2) = 30 = a[2]
```

### デリファレンスの図解

```
p が 2000 を保存している:

  p
┌──────┐          アドレス 2000
│ 2000 │ ────→   ┌────────┐
└──────┘          │   10   │
                  └────────┘
                   *p = 10

「p の中を見る」= 2000 という住所
「*p を見る」= 住所 2000 に実際に行って値 10 を読む
```

### swap 関数のメモリ図

```c
void swap(int *a, int *b) {
    int tmp = *a;
    *a = *b;
    *b = tmp;
}
```

```
呼び出し前:
main:  x=3 (アドレス:3000)   y=7 (アドレス:3004)

swap(&x, &y) を呼ぶ:
swap 内:
  a = 3000 (x のアドレス)
  b = 3004 (y のアドレス)

  tmp = *a = *(3000) = 3
  *a = *b → アドレス 3000 に *(3004)=7 を書く → x = 7
  *b = tmp → アドレス 3004 に 3 を書く → y = 3

呼び出し後:
main:  x=7   y=3   ← 入れ替わった！
```

## 7. よくある間違い

### 間違い1: `int *p` の `*` と `*p` の `*` を混同する

```c
// ❌ 混乱している例
int *p;      // 「p はポインタ」という宣言
*p = 5;      // 「p の指す先に 5 を書く」← p を初期化してないと危険！

// ✅ 正しい使い方
int x = 0;
int *p = &x;  // p を x のアドレスで初期化してから
*p = 5;       // x = 5 になる（安全）
```

**ルール:** ポインタを使う前に必ず有効なアドレスを代入する。

### 間違い2: 初期化していないポインタを使う

```c
// ❌ 致命的！
int *p;     // p の値はゴミ（ランダムなアドレス）
*p = 100;   // 存在しないアドレスに書き込む → クラッシュ

// ✅ 必ず初期化するか NULL に設定
int *p = NULL;
// または
int x = 0;
int *p = &x;
```

### 間違い3: `(*p)++` と `*p++` の違い

```c
int a[] = {10, 20, 30};
int *p = a;

// ① (*p)++ : p が指す値を 1 増やす（p は動かない）
(*p)++;  // a[0] = 11, p はまだ a[0] を指している

// ② *p++ : p を動かしてから前の位置の値を読む（ポストインクリメント）
// 演算子優先順位: ++ が * より先に評価される実装依存の動作に注意
// 試験では (*p)++ を使うほうが安全
```

**試験対策:** `(*p)++` は「値を増やす」、`p++` は「ポインタを進める」と覚える。

### 間違い4: ローカル変数のアドレスを返す（ダングリングポインタ）

```c
// ❌ 危険！
int* get_value(void) {
    int x = 42;   // ← x は get_value の中だけ存在する
    return &x;    // ← 関数が終わると x は消える！
}

int main() {
    int *p = get_value();
    printf("%d\n", *p);   // 未定義動作（クラッシュ or ゴミ値）
}
```

### 間違い5: `p[-1]` の危険性

```c
int a[] = {10, 20, 30};
int *p = a + 1;  // p は a[1] を指している

*(p + 1) = 30;  // a[2] → OK
*(p - 1) = 10;  // a[0] → OK（配列内なので安全）
p[-1]           // = *(p-1) = a[0] → 文法的にOKだが読みにくい
p[-2]           // = *(p-2) = a[-1] → 配列外！危険！
```

配列の範囲外アクセスは未定義動作です。

## 8. ミニクイズ

**Q1.** `int x = 10; int *p = &x;` のとき、`*p` の値は?

<details><summary>答え</summary>
`10` — p は x のアドレスを指しているので、*p は x の値を読む。
</details>

---

**Q2.** `int x = 5; int *p = &x; *p = 99;` の後、`x` の値は?

<details><summary>答え</summary>
`99` — `*p = 99` は「p が指す場所（= x のアドレス）に 99 を書く」なので、x が 99 になる。
</details>

---

**Q3.** 次のコードの出力は?

```c
int a[] = {1, 2, 3, 4, 5};
int *p = a + 2;
printf("%d %d\n", *p, *(p + 1));
```

<details><summary>答え</summary>
`3 4` — `a+2` は `a[2]` のアドレスなので `*p = 3`。`*(p+1)` は `a[3] = 4`。
</details>

---

**Q4.** `(*p)++` と `*p++` の違いを一言で言うと?

<details><summary>答え</summary>
`(*p)++` は「p が指す値を 1 増やす」。`*p++` は「p 自体を 1 進める（ポインタ移動）」。
</details>

---

**Q5.** `swap(int *a, int *b)` 関数を呼び出す正しい方法は?

```c
int x = 3, y = 7;
swap( _____ , _____ );
```

<details><summary>答え</summary>
`swap(&x, &y);` — ポインタを引数に取る関数には、変数のアドレス（`&変数名`）を渡す。
</details>

## 9. 例題

### 例題1: swap 関数（完全版 + トレース）

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int tmp = *a;   // tmp = a が指す値
    *a = *b;        // a の指す場所に b の値を書く
    *b = tmp;       // b の指す場所に tmp を書く
}

int main() {
    int x = 3, y = 7;
    printf("Before: x=%d y=%d\n", x, y);
    swap(&x, &y);
    printf("After:  x=%d y=%d\n", x, y);
    return 0;
}
```

**出力:**
```
Before: x=3 y=7
After:  x=7 y=3
```

**トレース（メモリ図）:**
```
swap 内:
  a = &x (例: アドレス 1000)
  b = &y (例: アドレス 1004)

  実行前: x=3, y=7
  tmp = *a = 3
  *a = *b   → x = 7   (アドレス1000 に 7 を書く)
  *b = tmp  → y = 3   (アドレス1004 に 3 を書く)
  実行後: x=7, y=3 ✓
```

### 例題2: ポインタで配列を走査して合計を求める

```c
#include <stdio.h>

int sum_array(int *p, int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += *(p + i);   // a[i] と等価
    }
    return total;
}

int main() {
    int data[] = {10, 20, 30, 40, 50};
    int s = sum_array(data, 5);
    printf("合計: %d\n", s);  // 合計: 150
    return 0;
}
```

**別の書き方（p++ を使う）:**
```c
int sum_array_v2(int *p, int n) {
    int total = 0;
    int *end = p + n;       // 配列の終端アドレス
    while (p < end) {
        total += *p;
        p++;                // ポインタを1つ進める
    }
    return total;
}
```

### 例題3: 文字列の逆順（2ポインタ法）

```c
#include <stdio.h>
#include <string.h>

void reverse(char *s) {
    char *left  = s;                  // 先頭を指すポインタ
    char *right = s + strlen(s) - 1; // 末尾を指すポインタ

    while (left < right) {
        char tmp = *left;
        *left  = *right;
        *right = tmp;
        left++;    // 左ポインタを右へ
        right--;   // 右ポインタを左へ
    }
}

int main() {
    char s[] = "hello";
    reverse(s);
    printf("%s\n", s);   // olleh
    return 0;
}
```

**2ポインタのメモリ図（"hello" を逆順に）:**
```
初期:   h  e  l  l  o  \0
        ↑              ↑
      left           right

1回目: swap(h, o) → o  e  l  l  h  \0
                      ↑        ↑
                    left     right

2回目: swap(e, l) → o  l  l  e  h  \0
                         ↑  ↑
                      left right（すれ違ったので終了）

結果: "olleh"
```
