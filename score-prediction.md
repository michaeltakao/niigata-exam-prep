> 作成日: 2026-06-19  
> データ根拠: R6・R7・R8 出題パターン分析

## 1. 試験スコアリングモデル

### 推定配点構造（R6〜R8 出題パターンより推算）

| 問題タイプ | 小問数 | 1問あたり | 推定合計 |
|---|---|---|---|
| 出力結果トレース | 3〜4問 | 8〜10点 | 30点 |
| 穴埋め（空欄補充） | 3〜5問 | 6〜8点 | 25点 |
| 関数実装 | 1〜2問 | 15〜20点 | 30点 |
| バグ修正 / 説明 | 1問 | 10〜15点 | 15点 |
| **合計** | | | **約100点** |

> **Assumption**: 実際の配点は非公開。上記は3年分の問題構成から推算した。

### 部分点ルール（推定）
- トレース問題: 変数1つ正解ごとに部分点あり（最終答えが違っても途中点）
- 穴埋め: 各空欄独立採点
- 実装問題: 動作しなくてもアルゴリズムの意図が読める場合に部分点
- → **途中式・コメントは必ず書くこと**

### トピック別配点目安

| トピッククラスタ | 推定配点 | Tier |
|---|---|---|
| 変数・型・printf/scanf | 10点 | Tier 1★ |
| 制御構造（if/for/while） | 20点 | Tier 1★ |
| 配列操作 | 20点 | Tier 1★ |
| 関数・再帰 | 25点 | Tier 1★/1 |
| ポインタ | 15点 | Tier 1 |
| 文字列・アルゴリズム | 10点 | Tier 1 |

## 2. 習熟度 → 期待スコア変換表

| 最高到達レベル | 習熟済みトピック | 期待スコア範囲 | 試験カバー率 |
|---|---|---|---|
| L0 (未経験) | なし | 0〜5点 | 〜5% |
| L1 (変数・型) | 変数宣言・printf・scanf | 5〜15点 | 〜15% |
| L2 (演算子) | + L1 + 演算子・式 | 10〜20点 | 〜20% |
| L3 (条件分岐) | + L1-L2 + if/switch | 15〜30点 | 〜35% |
| L4 (ループ) | + L1-L3 + for/while | 25〜45点 | 〜50% |
| L5 (配列) | + L1-L4 + 配列 | 35〜55点 | 〜60% |
| L6 (関数) | + L1-L5 + 関数・再帰 | 45〜65点 | 〜75% |

## 3. 弱点特定プロトコル

### ドリル成績からの弱点判定

各レベルのドリル（Easy 10問・Medium 10問・Hard 10問）を解いた後:

| 正答率 | 判定 | アクション |
|---|---|---|
| **≤ 40%** | ❌ 弱点（Weak） | 教科書を再読してから再挑戦 |
| **40〜70%** | △ 発展中（Developing） | 間違えた問題のエラー分類 → 矯正演習 |
| **≥ 70%** | ✅ 強い（Strong） | 次のレベルへ進む |

### 難易度別閾値

| 難易度 | Weak | Developing | Strong |
|---|---|---|---|
| Easy（10問） | ≤4問 | 5〜7問 | ≥8問 |
| Medium（10問） | ≤4問 | 5〜7問 | ≥8問 |
| Hard（10問） | ≤3問 | 4〜6問 | ≥7問 |

## 4. パーソナライズド推薦エンジン（ルールベース）

```
# 試験まで7日以内のルール

R01: IF weak_in(pointers) AND exam_in(7_days):
     → ポインタドリル Easy のみに集中（Medium/Hard は捨てる）
     → 「*p と p の違い」暗記カードを作る
     → 予測問題のポインタ問題だけ解く（実装は捨て、トレースに集中）

R02: IF weak_in(loops) AND exam_in(7_days):
     → L4 ドリル Easy 全問を再度解く（2周目）
     → トレース表を毎問作成する訓練
     → 過去問 R8 問1（ループトレース）を3回解く

R03: IF strong_in(all_L1_to_L5) AND weak_in(recursion) AND exam_in(7_days):
     → 再帰は「fact(4) と fib(5) のトレース」だけ完璧にする
     → 実装は捨て、出力結果問題と穴埋め問題だけ練習

# 試験まで3週間以上のルール

R04: IF strong_in(loops, arrays) AND weak_in(recursion):
     → L6 教科書「再帰」節を再読
     → fact → fib → (余力があれば) 二分探索の順で習得
     → 再帰ドリル Easy → Medium → Hard と段階的に進む

R05: IF weak_in(arrays) AND NOT_started(L5):
     → L5 教科書から始める（L6以上はまだやらない）
     → 1次元配列を完全習得してから2次元へ
     → 配列ドリル Easy 全問が Strong になるまでループ

R06: IF strong_in(all_L1_to_L7) AND exam_in(21_days):
     → 模擬試験 A → 分析 → 弱点補強 → 模擬試験 B の順
     → 1問あたりの解答時間を計測（目標: トレース5分・実装10分）

R07: IF pointers_score(easy) < 6 AND pointers_score(medium) >= 7:
     → Easy の失点はケアレスミス（不注意）の可能性が高い
     → diagnostic-checklist.md の「Careless Mistake」カテゴリを確認

R08: IF recursion_score(easy) >= 8 AND recursion_score(medium) < 5:
     → Medium 以上の問題で「複合再帰」（fib 等）に詰まっている
     → fib(5) の木展開を手書きで3回練習する

R09: IF all_easy_strong AND any_medium_weak:
     → アルゴリズムの設計力が不足（Logic Error カテゴリ）
     → 「コードを書く前に日本語で手順を書く」訓練
     → diagnostic-checklist.md の「Logic Error」矯正演習を実施

R10: IF syntax_errors_frequent:
     → C言語の文法ルール暗記が不完全
     → 「よくある文法エラー10パターン」を手書きで3回書く
     → コードを手書きしてから答え合わせする練習（試験が手書きのため重要）

R11: IF strong_in(L1_to_L6) AND NOT_started(L7_pointers):
     → ポインタは最も時間がかかるトピック
     → 試験まで14日以上あるなら学習開始、7日以内なら省略して基礎固め

R12: IF exam_in(3_days) AND score_projection < 50:
     → 「確実に取れる問題」に集中する戦略に切り替え
     → トレース問題（配点30点）に全時間を投入
     → 実装問題はドリルの Easy 問題のみ再確認
```
