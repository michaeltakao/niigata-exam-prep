## カテゴリ 1: 概念的誤解 (Conceptual Misunderstanding)

### 定義
「プログラムの動作モデル」自体を誤って理解しているために発生するエラー。コードを読む「脳内シミュレーター」が間違っている状態。

### 症状パターン（5種）
1. **値渡しを参照渡しと誤解**: `void f(int x){ x=100; }` を呼んだ後、呼び元の変数が 100 になると思う
2. **ループカウンタの終端誤解**: `for(i=0;i<5;i++)` が5回ではなく6回（i=0〜5）実行されると思う
3. **変数のスコープ誤解**: 関数内のローカル変数が関数外からもアクセスできると思う
4. **再帰の展開誤解**: `fact(3)` が `3*3*3=27` と計算されると思う（実際は `3*2*1=6`）
5. **`break` がプログラム全体を終了させると思う**: 実際は最内ループだけ脱出

### 根本原因
「プログラムがどう動くか」の精神的モデル（mental model）が不完全。教科書を読んだだけでコードを動かした経験がない。

### 矯正演習（3つ）
1. `void set100(int x){ x=100; } int n=0; set100(n); printf("%d",n);` — 出力前に予想して答え合わせする
2. 変数トレース表（ノートに書く）: ループごとに i, j, 合計 の値を1行1行手で記入する
3. `int fact(int n){ if(n<=1) return 1; return n*fact(n-1); }` で `fact(4)` を木構造で展開する

### 学習資料
`01_roadmap/learning-roadmap.md` の各レベルの「コア概念」と「自己テスト」

## カテゴリ 2: 論理エラー (Logic Error)

### 定義
文法は正しく、概念も理解しているが、アルゴリズムの設計ミスや条件の立て方を間違えているエラー。コードは動くが出力が違う。

### 症状パターン（5種）
1. **off-by-one**: `for(i=0;i<=n;i++)` → n+1 回実行（`<` と `<=` の混同）
2. **初期値エラー**: 最大値を `max=0` で初期化 → 全要素が負のとき誤答
3. **条件の向きミス**: 昇順ソートで `if(a[j] < a[j+1])` と書いて降順になる
4. **ループの内外配置ミス**: 合計変数を外で初期化すべきなのにループ内で初期化
5. **終了条件の論理ミス**: `while(i>0 || j>0)` と書くべきところを `&&` にする

### 根本原因
アルゴリズムを「自然言語で考えてからコードに落とす」工程が省略されている。また、境界値テスト（n=0, n=1, 最大値など）を精神的に試さない。

### 矯正演習（3つ）
1. バブルソートを「日本語で手順を書いてからコードに翻訳する」練習
2. `for(i=0;i<5;i++)` と `for(i=0;i<=5;i++)` でそれぞれ `printf("%d ",i)` を実行して回数を確認
3. 「最大値を求めるプログラム」を `{-5,-3,-1}` というテストケースで動かす

### 学習資料
`03_drills/L4_loops/medium.md`（ループの境界テスト問題）

## カテゴリ 3: 構文エラー (Syntax Error)

### 定義
C言語の文法規則に違反するエラー。コンパイル時に検出される（実行前にわかる）。

### 症状パターン（5種）
1. **セミコロン忘れ**: `int x = 5` → `error: expected ';'`
2. **型ミスマッチ**: `printf("%d", 3.14);` → 書式と型が一致しない（未定義動作）
3. **括弧の不対称**: `if (x > 0 {` → 括弧が閉じていない
4. **変数未宣言で使用**: `printf("%d", n);` — `n` を宣言前に使う
5. **文字列の引用符ミス**: `char c = "A";` — シングルクォートを使うべき（`'A'`）

### 根本原因
C言語の文法を「完璧には」覚えていないか、タイプミス。試験では手書きのため特に発生しやすい。

### 矯正演習（3つ）
1. 以下のコードのバグを全て見つけよ:  
   ```c
   int main() {
     int x = 10
     printf("%f", x);
     if (x > 5 {
       printf("big\n")
     }
   }
   ```
   （答え: セミコロン2か所、`%d`に修正、`)`追加）
2. `char c = "Z"; printf("%c", c);` — 何が間違い? （`"Z"` → `'Z'`）
3. C言語の予約語一覧（`int, double, char, if, for, while, return, void`）を10回書く

### 学習資料
`02_textbook/L01_variables-types.md` の書式指定子表を暗記する

## カテゴリ 4: ポインタ誤解 (Pointer Misunderstanding)

### 定義
ポインタとアドレスに関する概念的誤解。L7 専用の診断カテゴリ。「メモリ番地」の概念が固まっていない状態。

### 症状パターン（5種）
1. **`p` と `*p` の混同**: `printf("%d", p)` でアドレスが出力されているのに「値」と思う
2. **`&` の意味誤解**: `&a` が「a の値のコピー」と思う（実際は a のメモリアドレス）
3. **ポインタ算術の単位誤解**: `p+1` が「アドレスに 1 バイト加算」と思う（実際は `sizeof(型)` バイト加算）
4. **未初期化ポインタ使用**: `int *p; *p = 5;` — p が指すアドレスが不定
5. **`p++` と `(*p)++` の混同**: 前者はポインタ移動、後者は値変更

### 根本原因
「変数 = 箱」「ポインタ = 箱の住所」というメモリモデルを視覚的に理解していない。

### 矯正演習（3つ）
1. ノートに「メモリ図」を描く:  
   ```
   アドレス  値
   [1000]   5    ← int a = 5;
   [1004]  1000  ← int *p = &a;
   ```
   `*p`, `p`, `&a`, `&p` それぞれの値を記入する
2. `int a[]={10,20,30}; int *p=a; printf("%d %d %d",*p,*(p+1),*(p+2));` をトレースする
3. `swap(int *a, int *b)` を自力で実装し、`x=3,y=7` でテストする

### 学習資料
`02_textbook/L08_pointers.md` の「メモリ図」節を再読

## カテゴリ 5: 不注意ミス (Careless Mistake)

### 定義
概念も論理も正しく理解しているが、読み間違い・書き間違い・転記ミスによる失点。

### 症状パターン（5種）
1. **変数名の読み間違い**: `n1` と `n` を混同してトレース
2. **正負の符号見落とし**: `a[j] > a[j+1]` を `a[j] < a[j+1]` と読む
3. **初期値の転記ミス**: `int s = 1;` を `int s = 0;` と写して計算
4. **ループの終端を1つずらす**: `i<5` を `i<=5` と読む
5. **出力の改行見落とし**: `printf("%d\n")` の `\n` を忘れてスペースなし回答

### 根本原因
解答を急いで「流し読み」している。また、鉛筆での変数トレース表を作らずに暗算している。

### 矯正演習（3つ）
1. コードを指で一行一行指差しながらトレースする練習（声に出して変数の値を言う）
2. トレース表を必ずノートに書く（i=0のとき..., i=1のとき...）
3. 答えを出したら問題文のコードと照合して変数名・演算子を確認する

### 学習資料
`07_technique-guide/formula-first.md` の「変数トレース表」法
