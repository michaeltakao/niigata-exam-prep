# Formula-First テクニック — 試験問題の解き方

> このガイドは「C言語が初めて」の人が試験問題を解くための**実践的なテクニック集**です。
> コードを見て焦らずに、「手順通りにやれば解ける」という自信をつけることが目的です。

---

## 核心テクニック: Formula-First とは

**「コードを読む前に、プログラムが何をしているかを日本語（または数式）で書く」**

この1ステップを必ず先に行います。

### なぜこれが効くのか

- コードを見ると細部に引っかかって全体が見えなくなる
- 目的（何をするプログラムか）がわかれば、途中の計算が追いやすい
- 「何を求めているのか」がわかると、答えの妥当性チェックもできる

---

## テクニック 1: 変数トレース表

コードトレース問題（「実行結果を答えよ」）の必勝法です。

### 手順

1. プログラムに出てくる変数を全部リストアップする
2. 表を作る（変数が列、ステップが行）
3. コードを上から1行ずつ読んで、変数の値を更新する
4. `printf` が来たら、その時点の値を書き出す

### 例: 変数トレース表の作り方

**コード:**
```c
int a = 5;
int b = 3;
int sum = 0;
sum = a + b;
printf("%d\n", sum);
```

**トレース表:**

| ステップ | a | b | sum | 出力 |
|---|---|---|---|---|
| 初期化 | 5 | 3 | 0 | — |
| `sum = a + b` | 5 | 3 | 8 | — |
| `printf` | 5 | 3 | 8 | **8** |

→ **答え: 8**

---

## テクニック 2: ループの展開

ループが出てきたら「繰り返し」を全部テーブルに書き下します。

### 例: forループのトレース

**コード:**
```c
int i;
int sum = 0;
for (i = 1; i <= 4; i++) {
    sum = sum + i;
}
printf("%d\n", sum);
```

**ループ展開表:**

| ループ回 | i の値 | sum = sum + i の計算 | sum の新しい値 |
|---|---|---|---|
| 1回目 | 1 | 0 + 1 | 1 |
| 2回目 | 2 | 1 + 2 | 3 |
| 3回目 | 3 | 3 + 3 | 6 |
| 4回目 | 4 | 6 + 4 | 10 |
| 終了 | i=5, 5<=4が偽 | ループ終了 | — |

→ **答え: 10**

**チェック方法**: 「1から4の合計は 1+2+3+4=10」— 合っている！

---

## テクニック 3: 式の代入（Formula Substitution）

「この式は何を計算しているのか」を抽象式で書いてから数値を代入します。

### 例

**式:** `result = a * a + 2 * a * b + b * b`

**抽象化:** $(a + b)^2$ の展開式

**代入** (a=3, b=4): $(3+4)^2 = 49$

### なぜ抽象化するのか

- 長い式も「公式の形」が見えると一瞬で理解できる
- `a * a + 2 * a * b + b * b = (a+b)*(a+b)` と気づくと検算が楽

---

## パターンライブラリ — よく出るコード集

### パターン 1: 累積加算（ループで合計）

```c
sum = 0;
for (i = 0; i < n; i++) {
    sum = sum + arr[i];  /* または sum += arr[i]; */
}
```

**公式:** `sum = arr[0] + arr[1] + ... + arr[n-1]`

**意味:** 配列の全要素を足す

---

### パターン 2: 最大値・最小値スキャン

```c
max = arr[0];
for (i = 1; i < n; i++) {
    if (arr[i] > max) {
        max = arr[i];
    }
}
```

**公式:** 「今まで見た中で一番大きい値を `max` に覚えておく」

**トレース方法:** `max` の更新条件が満たされるときだけ `max` を書き直す

---

### パターン 3: 2変数のスワップ（交換）

```c
tmp = a;
a   = b;
b   = tmp;
```

**公式:** 「一時保管 `tmp` を経由して a と b を入れ替える」

**注意:** `a = b; b = a;` だけでは**入れ替えにならない**（`a` の元の値が消える）

---

### パターン 4: バブルソート（1パス）

```c
for (i = 0; i < n-1; i++) {
    for (j = 0; j < n-1-i; j++) {
        if (arr[j] > arr[j+1]) {
            tmp = arr[j];
            arr[j] = arr[j+1];
            arr[j+1] = tmp;
        }
    }
}
```

**公式:** 「隣り合う2要素を比べて大きい方を右へ送る。これを繰り返す。」

**トレース方法:** 1パスごとに配列の状態を書き下す

例 (arr = {3, 1, 4, 2}, 1パス目):
```
j=0: 3 > 1? → 交換 → {1, 3, 4, 2}
j=1: 3 > 4? → 交換なし → {1, 3, 4, 2}
j=2: 4 > 2? → 交換 → {1, 3, 2, 4}
1パス目終了: {1, 3, 2, 4}  ← 最大値 4 が末尾に
```

---

### パターン 5: 関数の値渡し

```c
void double_it(int x) {
    x = x * 2;
}

int main() {
    int a = 5;
    double_it(a);
    printf("%d\n", a);  /* 5 が出力される（aは変わらない） */
}
```

**公式:** 「関数に渡した変数は**コピー**が渡る。元の変数は変わらない。」

**注意:** これは編入試験の引っかけ問題としてよく出る！

---

### パターン 6: ポインタ渡し（上記の対比）

```c
void double_it(int *p) {
    *p = *p * 2;
}

int main() {
    int a = 5;
    double_it(&a);
    printf("%d\n", a);  /* 10 が出力される */
}
```

**公式:** 「`&a` = aのアドレス（住所）を渡す → 関数内で `*p` を使うと本物の `a` を書き換えられる」

---

### パターン 7: 再帰（factorial）

```c
int factorial(int n) {
    if (n == 0) return 1;      /* ベースケース */
    return n * factorial(n-1); /* 再帰呼び出し */
}
```

**公式:** `factorial(n) = n × (n-1) × ... × 1`

**トレース方法:** 呼び出しを積み上げる

```
factorial(3)
= 3 * factorial(2)
= 3 * 2 * factorial(1)
= 3 * 2 * 1 * factorial(0)
= 3 * 2 * 1 * 1
= 6
```

## 実践例題 1 — コードトレース問題

### 問題

次のプログラムの出力を答えよ。

```c
#include <stdio.h>

int main() {
    int i, j;
    for (i = 1; i <= 3; i++) {
        for (j = 1; j <= i; j++) {
            printf("*");
        }
        printf("\n");
    }
    return 0;
}
```

### Formula-First で解く

**Step 1: プログラムの目的を言語化する**
> 「外側のループで行番号 i (1〜3)、内側のループで `*` を i 個出力する。各行末に改行。」
> → 三角形の `*` パターンを出力するプログラム

**Step 2: ループ展開表**

| i | j の範囲 | 出力 |
|---|---|---|
| 1 | j=1 のみ | `*` + `\n` |
| 2 | j=1,2 | `**` + `\n` |
| 3 | j=1,2,3 | `***` + `\n` |

**答え:**
```
*
**
***
```

## 実践例題 2 — 関数実装問題

### 問題

配列 `arr` の要素数 `n` 個の中から最小値を返す関数を書け。

```c
int find_min(int arr[], int n) {
    /* ここを実装せよ */
}
```

### Formula-First で解く

**Step 1: 目的を言語化する**
> 「最初の要素を仮の最小値にして、後ろを1つずつ比べて、もっと小さければ更新する。」

**Step 2: パターンライブラリを適用**
> パターン 2（最大値スキャン）の `>` を `<` に変えて `min` に応用する。

**Step 3: コードを書く**

```c
int find_min(int arr[], int n) {
    int min = arr[0];    /* 最初の要素を仮の最小値にする */
    int i;
    for (i = 1; i < n; i++) {
        if (arr[i] < min) {
            min = arr[i];  /* もっと小さければ更新 */
        }
    }
    return min;
}
```

**Step 4: 手動テスト**

arr = {3, 1, 4, 1, 5}, n = 5 のとき:
- 初期: min = 3
- i=1: arr[1]=1 < 3 → min = 1
- i=2: arr[2]=4 < 1? → No
- i=3: arr[3]=1 < 1? → No
- i=4: arr[4]=5 < 1? → No
- return 1 ✓

## 実践例題 3 — バブルソートのトレース

### 問題

次のプログラムを実行したとき、配列 `arr` の各パス後の状態を全て答えよ。
初期状態: `arr = {5, 3, 8, 1}`

```c
int n = 4;
int i, j, tmp;
for (i = 0; i < n-1; i++) {
    for (j = 0; j < n-1-i; j++) {
        if (arr[j] > arr[j+1]) {
            tmp = arr[j];
            arr[j] = arr[j+1];
            arr[j+1] = tmp;
        }
    }
}
```

### 解答（パターン4を適用）

**パス 1 (i=0, j: 0→2):**
```
初期:  {5, 3, 8, 1}
j=0: 5>3? → 交換: {3, 5, 8, 1}
j=1: 5>8? → No:  {3, 5, 8, 1}
j=2: 8>1? → 交換: {3, 5, 1, 8}
パス1後: {3, 5, 1, 8}
```

**パス 2 (i=1, j: 0→1):**
```
j=0: 3>5? → No:  {3, 5, 1, 8}
j=1: 5>1? → 交換: {3, 1, 5, 8}
パス2後: {3, 1, 5, 8}
```

**パス 3 (i=2, j: 0→0):**
```
j=0: 3>1? → 交換: {1, 3, 5, 8}
パス3後: {1, 3, 5, 8}
```

**完成: {1, 3, 5, 8}** (昇順ソート完了)

## まとめ

1. **コードトレース問題** → 変数トレース表 + ループ展開表
2. **関数実装問題** → 目的を言語化 → パターンライブラリから選ぶ → 手動テスト
3. **ソート問題** → 1パスずつ配列の状態を書き下す
4. **わからなくなったら** → 「このプログラムは何をしているのか？」に戻る
