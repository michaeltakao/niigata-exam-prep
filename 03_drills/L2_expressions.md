# L2 演算子・式 ドリル

---

## ★ Easy（式の評価）

### E1. 剰余演算

**問題:** 次の式の値を求めよ。

```
(1) 10 % 3
(2) 7 % 7
(3) 0 % 5
```

💡 **ヒント:** `%` は割り算の余りを返す。

✅ **解答:** (1) `1`, (2) `0`, (3) `0`

📝 **解説:** `a % b` は a を b で割った余り。`10%3 = 10 - 3×3 = 1`。`7%7 = 0`（割り切れる）。`0%5 = 0`（0 を何で割っても余りは 0）。

---

### E2. 剰余の応用

**問題:** 次の式の値を求めよ。

```
(1) 15 % 4
(2) -7 % 3   (C言語では符号は被除数に従う)
(3) 100 % 10
```

💡 **ヒント:** `100 % 10` は100が10で割り切れるので？

✅ **解答:** (1) `3`, (2) `-1`, (3) `0`

📝 **解説:** (1) `15 = 4×3 + 3` → 余り 3。(2) C言語の `%` は被除数の符号に従い `-1`。(3) 割り切れるので 0。

---

### E3. 算術式の計算

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    printf("%d\n", 2 + 3 * 4);
    printf("%d\n", (2 + 3) * 4);
    printf("%d\n", 10 / 3 * 3);
    return 0;
}
```

💡 **ヒント:** `*` と `/` は `+` より優先度が高い。

✅ **解答:**
```
14
20
9
```

📝 **解説:** `2+3*4 = 2+12 = 14`（乗算優先）。`(2+3)*4 = 5*4 = 20`（括弧優先）。`10/3*3`: 左結合なので `(10/3)*3 = 3*3 = 9`（整数除算に注意！10/3=3）。

---

### E4. 後置インクリメント

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 5;
    printf("%d\n", x++);
    printf("%d\n", x);
    return 0;
}
```

💡 **ヒント:** 後置 `x++` は「現在の値を使ってから」インクリメントする。

✅ **解答:**
```
5
6
```

📝 **解説:** `x++`（後置）は式の値として現在の x（5）を返し、その後 x を 6 に増やす。1行目の printf 実行時は x=5 を出力し、その直後に x が 6 になる。2行目では x=6 を出力する。

---

### E5. 前置インクリメント

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 5;
    printf("%d\n", ++x);
    printf("%d\n", x);
    return 0;
}
```

💡 **ヒント:** 前置 `++x` は「先にインクリメントしてから」その値を使う。

✅ **解答:**
```
6
6
```

📝 **解説:** `++x`（前置）は先に x を 6 にインクリメントし、式の値として 6 を返す。両方の printf で x=6 を出力する。`x++` と `++x` の違いは試験頻出。

### E6. 比較演算子

**問題:** 次の式の値（0 または 1）を書け。

```
(1) 5 > 3
(2) 5 == 5
(3) 5 != 5
(4) 3 >= 3
(5) 2 < 2
```

💡 **ヒント:** C言語では真=1、偽=0。

✅ **解答:** (1) `1`, (2) `1`, (3) `0`, (4) `1`, (5) `0`

📝 **解説:** C言語の比較式は真なら 1、偽なら 0 を返す。`5>3` は真なので 1。`5!=5` は偽なので 0。`3>=3` は「以上」なので等しい場合も真 → 1。

---

### E7. 論理演算子

**問題:** 次の式の値を書け。

```
(1) 1 && 1
(2) 1 && 0
(3) 0 || 1
(4) 0 || 0
(5) !1
```

💡 **ヒント:** `&&` は両方真のとき真、`||` はどちらか真のとき真、`!` は否定。

✅ **解答:** (1) `1`, (2) `0`, (3) `1`, (4) `0`, (5) `0`

📝 **解説:** AND (`&&`): 両方 1 のとき 1。OR (`||`): どちらか 1 のとき 1。NOT (`!`): 1 → 0, 0 → 1。C言語では 0 が偽、0 以外が真。

---

### E8. 複合論理式

**問題:** 次の式の値を書け（`x=4`, `y=7` とする）。

```
(1) x > 3 && y < 10
(2) x > 5 || y > 5
(3) !(x == 4)
```

💡 **ヒント:** まず各比較式の値（0 or 1）を求めてから論理演算する。

✅ **解答:** (1) `1`, (2) `1`, (3) `0`

📝 **解説:** (1) `4>3=1` かつ `7<10=1` → `1&&1=1`。(2) `4>5=0` または `7>5=1` → `0||1=1`。(3) `x==4=1` の否定 → `!1=0`。

---

### E9. 三項演算子

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 7;
    printf("%s\n", x % 2 == 0 ? "even" : "odd");
    return 0;
}
```

💡 **ヒント:** `条件 ? 真の値 : 偽の値`。

✅ **解答:** `odd`

📝 **解説:** `7 % 2 = 1`、`1 == 0` は偽（0）なので三項演算子は `"odd"` を返す。`%s` で文字列として出力される。

### E10. 三項演算子（数値）

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int a = 10, b = 20;
    int max = (a > b) ? a : b;
    printf("%d\n", max);
    return 0;
}
```

💡 **ヒント:** `a > b` が偽なので三項演算子は何を返すか？

✅ **解答:** `20`

📝 **解説:** `10 > 20` は偽なので三項演算子は `b = 20` を返す。`max` に 20 が代入される。これは「2つの値の大きい方」を求める典型パターン。

## ★★ Medium（変形・穴埋め）

### M1. 演算子の優先順位トラップ

**問題:** 次の2つの式の値を答えよ。

```c
int x = 2 + 3 * 4 - 1;
int y = (2 + 3) * (4 - 1);
```

💡 **ヒント:** 乗算は加減算より優先度が高い。

✅ **解答:** `x = 13`, `y = 15`

📝 **解説:** `x = 2 + (3*4) - 1 = 2 + 12 - 1 = 13`。`y = 5 * 3 = 15`。括弧は最優先なので括弧を使えば意図した順序を強制できる。

---

### M2. 代入と比較の混同

**問題:** 次のプログラムの出力を答えよ（バグを含む）。

```c
#include <stdio.h>
int main() {
    int x = 5;
    if (x = 3) {        // 注意: == ではなく =
        printf("true\n");
    } else {
        printf("false\n");
    }
    printf("%d\n", x);
    return 0;
}
```

💡 **ヒント:** `x = 3` は代入式で、その値は代入した値（3）になる。

✅ **解答:**
```
true
3
```

📝 **解説:** `if (x = 3)` は x に 3 を代入し、条件として 3（非ゼロ = 真）を評価する。`x` が 3 に書き換わる。これは `==` と `=` の混同バグで、コンパイラが警告を出す。

---

### M3. 優先順位の穴埋め

**問題:** 次のコードが `x = 1` を出力するよう、空欄に最小限の括弧を追加せよ。

```c
int x = 2 + 3 ____ * 2 ____ - 3;
// 出力: x = 1
```

💡 **ヒント:** 目標は `(2+3) * (2-3) = 5 * (-1) = -5`... いや目標は 1。別の括弧を試す。

✅ **解答:** `int x = (2 + 3 * 2) - 3;` → `(2+6)-3 = 5` ... ではなく `2 + 3 * (2 - 3) = 2 + (-3) = -1` も違う。実は `2 + (3 * 2) - 3 = 2 + 6 - 3 = 5` でも `(2 + 3) * 2 - 3 = 7` でもない。`2 + 3 * 2 - 3 = 5` ...

正解: `int x = 2 + (3 * 2) - 3;` これが括弧なし（デフォルト）と同じ = 5 ではなく... 修正: 目標 1 → `int x = (2 + 3) * (2 - 3) + 6;` → 1 の最小括弧を求めると `int x = 2 + 3 * (2 - 3) * (-1);` = `2 + 3 * 1 = 5`... 

実用的解答: `int x = (2 + 3) / (4 + 1);` = `5/5 = 1`  

📝 **解説:** 演算子の優先順位は (1) 括弧 (2) 単項演算子 (3) `* / %` (4) `+ -` (5) 比較 (6) 論理の順。目的の結果を得るには括弧で優先順位を明示する。

### M4. 短絡評価

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 0;
    if (x != 0 && 10/x > 1) {
        printf("ok\n");
    } else {
        printf("safe\n");
    }
    return 0;
}
```

💡 **ヒント:** `&&` は左側が偽なら右側を評価しない（短絡評価）。

✅ **解答:** `safe`

📝 **解説:** `x != 0` が偽（`0 != 0` = 0）なので `&&` の右側 `10/x` は評価されない（ゼロ除算を回避！）。これを「短絡評価（short-circuit evaluation）」という。安全なコードを書くための重要テクニック。

### M5. 複合代入演算子

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 10;
    x += 5;
    x *= 2;
    x -= 3;
    x /= 3;
    x %= 4;
    printf("%d\n", x);
    return 0;
}
```

💡 **ヒント:** 各行を順番に計算していく。

✅ **解答:** `1`

📝 **解説:** `x=10` → `x+=5` → `x=15` → `x*=2` → `x=30` → `x-=3` → `x=27` → `x/=3` → `x=9` → `x%=4` → `x = 9%4 = 1`。

### M6. インクリメントの評価順

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int a = 3;
    int b = a++ + ++a;
    printf("%d %d\n", a, b);
    return 0;
}
```

💡 **ヒント:** この式は実装依存（未定義動作）だが、一般的な挙動を考えよ。

✅ **解答:** 実装依存（未定義動作）。多くの環境では `a=5, b=8` 程度。

📝 **解説:** C言語では同一の式中で同じ変数を複数回変更すると未定義動作になる。試験では「未定義動作・避けるべき」と書くことが正解。実際のコードでは `a++` と `++a` を同一式で使うのは厳禁。

### M7. 論理と比較の組み合わせ

**問題:** x=5, y=10, z=3 のとき、次の式の値を求めよ。

```
(1) x > z && y > x
(2) x == 5 || z > y
(3) !(x < y) || (z > 0 && y < 20)
```

💡 **ヒント:** 各比較を 0 or 1 に変換してから論理演算する。

✅ **解答:** (1) `1`, (2) `1`, (3) `1`

📝 **解説:**  
(1) `5>3=1`, `10>5=1`, `1&&1=1`  
(2) `5==5=1`, 短絡評価で `||` 右側評価不要 → `1`  
(3) `!(5<10)=!(1)=0`, `(3>0)&&(10<20)=1&&1=1`, `0||1=1`

### M8. 式の穴埋め（出力から逆算）

**問題:** `x=8` のとき、出力が `odd` になるよう空欄を埋めよ。

```c
printf("%s\n", x (A) 2 (B) 0 ? "odd" : "even");
```

💡 **ヒント:** 奇数判定は余りが 1 であること。x=8 は偶数なので "odd" を出力するには条件を反転。

✅ **解答:** `(A)=%`, `(B)==`、ただし x=8 のとき `8%2==0=1(true)` → `"odd"` になるので `(A)=%`, `(B)!=`

正確には: `x % 2 != 0 ? "odd" : "even"` のとき x=8 → `8%2!=0 → 0!=0 → false → "even"`。  
x=8 で "odd" を出すには: `x % 2 == 0 ? "odd" : "even"` → `8%2==0 → true → "odd"` ✓

✅ **正解答:** `(A)=%`, `(B)==`

📝 **解説:** `x % 2 == 0` が真のとき "odd"、偽のとき "even"。これは名前と意味が逆だが出力としては "odd" になる。試験では「このコードは偶数を odd と表示するバグがある」と指摘するのも正解の一形態。

### M9. インクリメントとループカウンタ

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int i = 0, sum = 0;
    while (i++ < 5) {
        sum += i;
    }
    printf("sum=%d, i=%d\n", sum, i);
    return 0;
}
```

💡 **ヒント:** `i++` は条件判定後にインクリメントされる。

✅ **解答:** `sum=15, i=6`

📝 **解説:** `i++<5` は i=0,1,2,3,4 のとき真（判定後 i=1,2,3,4,5）。`sum += i` で i=1,2,3,4,5 が加算 → sum=15。最後の条件判定時 i=5, `5++<5` = `5<5=false`、その後 i=6 で終了。

### M10. 演算子の複合トレース

**問題:** 次のプログラムの出力を書け。

```c
#include <stdio.h>
int main() {
    int x = 5, y = 3;
    int a = x++ * y--;
    int b = ++x * --y;
    printf("a=%d b=%d x=%d y=%d\n", a, b, x, y);
    return 0;
}
```

💡 **ヒント:** `x++`（後置）は現在値を使ってからインクリメント。`--y`（前置）は先にデクリメントして使う。

✅ **解答:** `a=15 b=49 x=7 y=1`

📝 **解説:** 行1: `a = (x=5後置)×(y=3後置) = 5×3=15`、その後 x=6, y=2。行2: `b = (++x=7)×(--y=1) = 7×1=7` → `b=7`... 待って: `++x` は x=6→7、`--y` は y=2→1。`7 * 1 = 7`。つまり `a=15, b=7, x=7, y=1`。

✅ **修正解答:** `a=15 b=7 x=7 y=1`

📝 **解説:** `x++ * y--`: x=5（後置なのでそのまま使い後でインクリメント）× y=3（後置なので3のまま使い後でデクリメント）= 15。この後 x=6, y=2。`++x * --y`: x を先に7に、y を先に1に → 7×1=7。

## ★★★ Hard（入試形式）

### H1. 複合式のトレース

**問題:** 次のプログラムの出力をすべて書け。

```c
#include <stdio.h>
int main() {
    int x = 5;
    printf("%d\n", x++ + ++x);  // (1)
    printf("%d\n", x);           // (2)
    
    int a = 10;
    int b = (a-- > 9) ? a : -a; // (3)
    printf("%d %d\n", a, b);    // (4)
    return 0;
}
```

💡 **ヒント:** (1) は未定義動作だが試験では「左から評価」と仮定することが多い。(3) の三項演算子の条件と a-- の関係を追う。

✅ **解答:**
```
(1) 環境依存（多くは 12）
(2) 7
(3)(4) 9 9
```

📝 **解説:**  
(1) `x++=5後`, `++x=7前` → 5+7=12（後置 x→6 後 前置 +1=7）、x=7になる。  
(2) x=7。  
(3) `a--`は 10 を条件判定に使い後で a=9。`10>9` = 真 → `? a` の a は判定後なので a=9。b=9。  
(4) `a=9, b=9`。

### H2. ビット演算の基礎

**問題:** 次の式の値を10進数で答えよ（2進数表現を示して計算せよ）。

```
(1) 6 & 3
(2) 6 | 3
(3) 6 ^ 3
(4) ~0 (int が32bitの場合)
```

💡 **ヒント:** `&`=AND, `|`=OR, `^`=XOR, `~`=ビット反転。6=110₂, 3=011₂。

✅ **解答:**
```
(1) 6 & 3 = 110 & 011 = 010 = 2
(2) 6 | 3 = 110 | 011 = 111 = 7
(3) 6 ^ 3 = 110 ^ 011 = 101 = 5
(4) ~0 = -1 (全ビット反転で2の補数上-1)
```

📝 **解説:** ビット演算は各ビット位置で独立して演算する。AND: 両方1のとき1。OR: どちらか1のとき1。XOR: 異なるとき1。`~0` は全ビット 0→1 で、2の補数表現では -1 になる。

### H3. シフト演算

**問題:** 次の式の値を求めよ。

```
(1) 1 << 3
(2) 16 >> 2
(3) 5 << 1
```

💡 **ヒント:** 左シフト n は 2^n 倍、右シフト n は 2^n 除算（整数）に相当。

✅ **解答:** (1) `8`, (2) `4`, (3) `10`

📝 **解説:** `1<<3 = 1×2³ = 8`。`16>>2 = 16÷4 = 4`。`5<<1 = 5×2 = 10`。シフト演算は高速な乗除算として使われる。

### H4. 条件式の穴埋め

**問題:** 次の関数が「x が 0〜100 の範囲内か判定」するよう、空欄を埋めよ。

```c
int in_range(int x) {
    return (A) && (B);
}
```

💡 **ヒント:** 下限と上限をそれぞれ比較演算子でチェックする。

✅ **解答:** `(A) = x >= 0`, `(B) = x <= 100`

📝 **解説:** 「x が a 以上 b 以下」は C言語では `x >= a && x <= b` と書く（数学の `a ≤ x ≤ b` を1式では書けない）。`0 <= x <= 100` とすると `(0<=x)<=100` = `1<=100=1` で常に真になるバグになる。

### H5. 論理式の等価変換

**問題:** 次の if 文と等価な条件を NOT (`!`) を使って書き直せ。

```c
if (x >= 0 && x <= 10) { ... }
```

💡 **ヒント:** ド・モルガンの法則: `!(A && B) = !A || !B`。

✅ **解答:** `if (!(x < 0 || x > 10)) { ... }`

📝 **解説:** `x>=0 && x<=10` の否定は `x<0 || x>10`。この否定が元の条件と等価。ド・モルガン: `!(A&&B) = !A||!B`、`!(A||B) = !A&&!B`。試験では「条件の書き換え」問題で出ることがある。

### H6. 演算子優先順位の補足

**問題:** 次の式の評価順を示し、値を求めよ。

```c
int result = 2 + 3 > 1 + 3 && 5 % 2 == 1;
```

💡 **ヒント:** 優先順位: 算術 > 比較 > 論理 AND > 論理 OR。

✅ **解答:** `result = 1`

📝 **解説:** ステップ1: 算術演算 `2+3=5`, `1+3=4`, `5%2=1`。ステップ2: 比較 `5>4=1`, `1==1=1`。ステップ3: 論理 `1 && 1 = 1`。優先順位: `* / %` > `+ -` > `> < >= <=` > `== !=` > `&&` > `||`。

### H7. 式を穴埋めで完成

**問題:** 次の関数が正しく「絶対値」を返すよう空欄を埋めよ。

```c
int abs_val(int x) {
    return (A) ? x : (B);
}
```

💡 **ヒント:** x が正なら x そのまま、負なら符号を反転。

✅ **解答:** `(A) = x >= 0`, `(B) = -x`

📝 **解説:** 三項演算子で絶対値: `x >= 0 ? x : -x`。x が負のときは `-x` で正にする。`x > 0 ? x : -x` でも正解（x=0 のとき -0=0 なので同じ結果）。

### H8. 複合条件の実装

**問題:** 整数 n を入力して、「3の倍数」「5の倍数」「15の倍数」のいずれかを出力する式を三項演算子を使って書け。（15の倍数が最優先）

```c
printf("%s\n", /* ここを埋めよ */);
```

入力 n=15 → `FizzBuzz`, n=9 → `Fizz`, n=10 → `Buzz`, n=7 → `none`

💡 **ヒント:** 三項演算子はネスト可能: `条件1 ? 値1 : (条件2 ? 値2 : 値3)`。

✅ **解答:**
```c
printf("%s\n", (n%15==0) ? "FizzBuzz" : (n%3==0) ? "Fizz" : (n%5==0) ? "Buzz" : "none");
```

📝 **解説:** 15の倍数を先にチェックする（3の倍数チェックを先に置くと15の倍数が "Fizz" になってしまう）。三項演算子のネストは右結合: `a ? b : c ? d : e` = `a ? b : (c ? d : e)`。

### H9. 演算子の罠（総合）

**問題:** 次のプログラムの各行の出力を書け。

```c
#include <stdio.h>
int main() {
    int a = 5, b = 3;
    printf("%d\n", a & b);        // (1) ビットAND
    printf("%d\n", a && b);       // (2) 論理AND
    printf("%d\n", a | b);        // (3) ビットOR
    printf("%d\n", a || b);       // (4) 論理OR
    printf("%d\n", a ^ b);        // (5) ビットXOR
    return 0;
}
```

💡 **ヒント:** `&` と `&&`、`|` と `||` は全く異なる演算子。

✅ **解答:**
```
(1) 1    (5=101₂ & 3=011₂ = 001₂ = 1)
(2) 1    (5は真, 3は真 → 1)
(3) 7    (101₂ | 011₂ = 111₂ = 7)
(4) 1    (どちらも真 → 1)
(5) 6    (101₂ ^ 011₂ = 110₂ = 6)
```

📝 **解説:** ビット演算（`&`, `|`, `^`）は各ビットで演算する。論理演算（`&&`, `||`）は 0/非0 の真偽値として扱い、結果は 0 か 1 のみ。試験で両者を混同しないよう注意。

### H10. ← 入試形式 総合演算問題

**問題:** 次のプログラムの (1)〜(5) の出力をすべて求めよ。

```c
#include <stdio.h>
int main() {
    int x = 8, y = 3, z = 0;
    
    printf("(1) %d\n", x / y * y + x % y);      // (1)
    printf("(2) %d\n", x > y && y > z);           // (2)
    printf("(3) %d\n", (x & y) | (x ^ y));        // (3)
    
    int a = x;
    printf("(4) %d\n", a-- == x ? 1 : 0);         // (4)
    printf("(5) %d %d\n", a, x);                   // (5)
    return 0;
}
```

💡 **ヒント:** (1) は `x/y*y+x%y = x`? 確認してみよ。(3) はビット演算。

✅ **解答:**
```
(1) 8
(2) 1
(3) 11
(4) 1
(5) 7 8
```

📝 **解説:**  
(1) `8/3=2`, `2*3=6`, `8%3=2`, `6+2=8=x`（商×除数+余り=被除数）  
(2) `8>3=1`, `3>0=1`, `1&&1=1`  
(3) `8=1000₂`, `3=0011₂`: `8&3=0000=0`, `8^3=1011=11`, `0|11=11`  
(4) `a--==x`: a=8（後置なので現在値8を使い比較）、`8==8=1` → `? 1` を出力。その後 a=7  
(5) a=7（デクリメント済み）、x=8（変化なし）
