### 大問 I 解答

#### (1) FizzBuzz 実行結果

i が 1〜20 のうち、3の倍数・5の倍数・両方の倍数を出力:

```

Buzz
Fizz
FizzBuzz

Fizz

```

詳細:
- i=3: Fizz, i=5: Buzz, i=6: Fizz, i=9: Fizz, i=10: Buzz
- i=12: Fizz, i=15: FizzBuzz, i=18: Fizz, i=20: Buzz
- i=1,2,4,7,8,11,13,14,16,17,19 → 条件に該当せず（何も出力しない）
- 合計: Fizz=7, Buzz=4, FizzBuzz=1, count=12... 

再計算:
- 3の倍数（15を除く）: 3,6,9,12,18 → 5個（Fizz）
- 5の倍数（15を除く）: 5,10,20 → 3個（Buzz）  

  ※20 は 3の倍数ではないので Buzz。
- 3かつ5の倍数: 15 → 1個（FizzBuzz）
- count = 5+3+1 = **9**

### 大問 III 解答

#### (1) 各 printf の出力

配列 `a[] = {10, 20, 30, 40, 50}`, `p = a+2` → `p` は `a[2]` を指す（値 30）。

- `(1)` → `30`（`*p = a[2] = 30`）
- `(2)` → `70`（`*(p-1) + *(p+1) = a[1] + a[3] = 20 + 40 = 60`）

  **注意**: 再確認 → `20 + 40 = 60` → `(2) 60`
- `(3)` → `10`（`p[-2] = *(p-2) = a[0] = 10`）
- `(4)` → `1024`（`power(2,10) = 2^10 = 1024`）
- `(5)` → `NIIGATA`（to_upper で全文字を大文字に変換）

訂正版:
```
(1) 30
(2) 60
(3) 10
(4) 1024
(5) NIIGATA
```

#### (2) to_upper の空欄

- **(A)** `exp - 1`（指数を1減らして再帰）
- **(B)** `*s >= 'a' && *s <= 'z'`（小文字かどうかの判定）
- **(C)** `32`（小文字 → 大文字への変換: `'a' - 'A' = 97 - 65 = 32`）

完成形:
```c
void to_upper(char *s) {
    while (*s != '\0') {
        if (*s >= 'a' && *s <= 'z') {
            *s = *s - 32;
        }
        s++;
    }
}
```

#### (3) power の反復版

```c
int power_iter(int base, int exp) {
    int result = 1;
    int i;
    for (i = 0; i < exp; i++) {
        result *= base;
    }
    return result;
}
```

確認: `power_iter(2, 10)` → result = 1×2×2×…×2（10回）= 1024 ✓

#### (4) 改善案

**改善点**: `power` 関数に負の指数 `exp < 0` への対処がない。また、`exp=0` は正しく処理されるが、それ以外の境界値（`base=0` など）への対処がない。

```c
int power(int base, int exp) {
    if (exp < 0) {
        fprintf(stderr, "Error: negative exponent\n");
        return -1;
    }
    if (exp == 0) return 1;
    return base * power(base, exp - 1);
}
```

または、`to_upper` において `'A'〜'Z'` のリテラル依存を `toupper()` 標準関数に置き換えるのも改善案として有効。

```c
#include <ctype.h>
void to_upper(char *s) {
    while (*s != '\0') {
        *s = (char)toupper((unsigned char)*s);
        s++;
    }
}
```
