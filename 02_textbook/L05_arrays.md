# L05 配列 — 同じ型のデータをまとめて管理する

> **重要度**: Tier 1 ★★★ — R6・R7・R8 全年で出題。バブルソートとセットで最重要。

---

## 1. コンセプト概要

**配列（Array）** とは、同じ型のデータを連続して並べたものです。

```
変数が1つの場合:
  score = 85   ← 1人分しか入らない

配列なら5人分まとめて管理:
  scores[0]=85  scores[1]=92  scores[2]=78  scores[3]=90  scores[4]=65
```

配列の特徴：
- **インデックス（添字）は 0 始まり**（最初の要素は `a[0]`、Nこあれば最後は `a[N-1]`）
- 同じ型のデータだけ格納できる（int の配列に double は入らない）
- ループと組み合わせて使うのが基本パターン

---

## 2. なぜ必要か

**配列なしの場合**（100人の点数を管理）:
```c
int score0 = 85;
int score1 = 92;
int score2 = 78;
// ... 100個の変数が必要！
```

**配列を使った場合**:
```c
int scores[100];   // 100人分まとめて確保
for (int i = 0; i < 100; i++) {
    scanf("%d", &scores[i]);  // ループで一括入力
}
```

データが増えても、配列サイズを変えるだけで対応できます。

## 3. 現実世界のアナロジー

### 配列 → 教室の「出席番号」

```
クラス: scores[]
┌────┬────┬────┬────┬────┐
│ 85 │ 92 │ 78 │ 90 │ 65 │
└────┴────┴────┴────┴────┘
  [0]  [1]  [2]  [3]  [4]
   ↑                    ↑
 1番目の生徒          5番目の生徒
（出席番号0）        （出席番号4）
```

- 「出席番号3番の点数」→ `scores[3]`（= 90）
- 先生（プログラム）が出席番号順にループして全員の点数を確認するのがループ+配列

### 2次元配列 → 席替えした教室（行と列）

```
int classroom[3][4]:   // 3行 × 4列

      列0   列1   列2   列3
行0 [  1] [  2] [  3] [  4]
行1 [  5] [  6] [  7] [  8]
行2 [  9] [ 10] [ 11] [ 12]

classroom[1][2] = 7  （1行目、2列目の生徒）
```

## 4. 構文説明

### 4-1. 宣言

```c
int a[5];            // 要素5個のint配列（未初期化）
double b[10];        // 要素10個のdouble配列
char c[20];          // 要素20個のchar配列（文字列用）
```

### 4-2. 初期化

```c
// 方法1: 宣言と同時に初期化（サイズ自動計算）
int a[] = {10, 20, 30, 40, 50};   // サイズ5と自動推論

// 方法2: サイズを指定して初期化
int b[5] = {10, 20, 30, 40, 50};

// 方法3: 一部だけ初期化（残りは0）
int c[5] = {1, 2};   // c = {1, 2, 0, 0, 0}

// 方法4: 全要素0に初期化
int d[5] = {0};      // d = {0, 0, 0, 0, 0}
```

### 4-3. 要素へのアクセス

```c
int a[] = {10, 20, 30, 40, 50};

printf("%d\n", a[0]);   // 10（最初の要素）
printf("%d\n", a[4]);   // 50（最後の要素）

a[2] = 99;              // 3番目の要素を変更
printf("%d\n", a[2]);   // 99
```

### 4-4. 配列の要素数を取得する方法

```c
int a[] = {10, 20, 30, 40, 50};
int n = sizeof(a) / sizeof(a[0]);   // 5
// sizeof(a) = 20バイト、sizeof(a[0]) = 4バイト → 20/4 = 5
```

### 4-5. 配列 + for ループ（基本パターン）

```c
int a[] = {10, 20, 30, 40, 50};
int n = 5;

// 全要素を出力
for (int i = 0; i < n; i++) {
    printf("a[%d] = %d\n", i, a[i]);
}
```

### 4-6. 2次元配列

```c
int matrix[3][4];       // 3行4列

// 初期化
int m[2][3] = {
    {1, 2, 3},          // 0行目
    {4, 5, 6}           // 1行目
};

printf("%d\n", m[0][2]);  // 3（0行2列）
printf("%d\n", m[1][1]);  // 5（1行1列）
```

## 5. ステップ実行トレース

```c
#include <stdio.h>
int main() {
    int scores[] = {85, 92, 78, 90, 65};
    int n = 5;
    int sum = 0;
    int max = scores[0];   // まず先頭を最大値と仮定

    for (int i = 0; i < n; i++) {
        sum += scores[i];
        if (scores[i] > max) {
            max = scores[i];
        }
    }
    printf("合計: %d\n", sum);
    printf("最大: %d\n", max);
    return 0;
}
```

### 完全トレース表

| i | scores[i] | sum（前） | sum（後） | scores[i]>max? | max（後） |
|---|---|---|---|---|---|
| 初期 | — | 0 | 0 | — | 85（=scores[0]） |
| 0 | 85 | 0 | 85 | 85>85? **false** | 85 |
| 1 | 92 | 85 | 177 | 92>85? **true** | **92** |
| 2 | 78 | 177 | 255 | 78>92? false | 92 |
| 3 | 90 | 255 | 345 | 90>92? false | 92 |
| 4 | 65 | 345 | 410 | 65>92? false | 92 |

**出力**:
```
合計: 410
最大: 92
```

## 6. 視覚的メモリ図

### 配列のメモリ上の配置

```
int a[] = {10, 20, 30, 40, 50};

メモリアドレス（概念図）:
  1000  1004  1008  1012  1016
  ┌────┬────┬────┬────┬────┐
  │ 10 │ 20 │ 30 │ 40 │ 50 │
  └────┴────┴────┴────┴────┘
  a[0] a[1] a[2] a[3] a[4]

各 int は 4バイト、配列は連続したメモリ領域に格納される
```

### バブルソートの動きを視覚化

```
初期: [64, 34, 25, 12, 22]

パス1:
  64>34 → swap: [34, 64, 25, 12, 22]
  64>25 → swap: [34, 25, 64, 12, 22]
  64>12 → swap: [34, 25, 12, 64, 22]
  64>22 → swap: [34, 25, 12, 22, 64] ← 64が末尾に確定！

パス2:
  34>25 → swap: [25, 34, 12, 22, 64]
  34>12 → swap: [25, 12, 34, 22, 64]
  34>22 → swap: [25, 12, 22, 34, 64] ← 34が確定
  ...
```

### 2次元配列の視覚化

```
int m[3][3] = {{1,2,3},{4,5,6},{7,8,9}};

     j=0  j=1  j=2
i=0 [ 1 ][ 2 ][ 3 ]
i=1 [ 4 ][ 5 ][ 6 ]
```

## 7. よくある間違い

### 間違い1: 範囲外アクセス（最も危険なバグ）

```c
int a[5] = {10, 20, 30, 40, 50};

// ❌ 間違い: a[5] は存在しない（インデックスは 0〜4）
printf("%d\n", a[5]);   // 未定義動作！予測不能な値が出る

// ✅ 正解: 最後の要素は a[4]
printf("%d\n", a[4]);   // 50
```

**チェック方法**: 宣言が `int a[N]` なら、有効インデックスは `0` から `N-1` まで。

### 間違い2: ループ条件のミス（i <= n は危険）

```c
int a[5] = {10, 20, 30, 40, 50};
int n = 5;

// ❌ 間違い: i <= 5 だと a[5] にアクセスしてしまう
for (int i = 0; i <= n; i++) {
    printf("%d\n", a[i]);   // i=5 のとき範囲外！
}

// ✅ 正解: i < n
for (int i = 0; i < n; i++) {
    printf("%d\n", a[i]);
}
```

### 間違い3: 最大値の初期化ミス

```c
// ❌ 間違い: 0 で初期化すると、全要素が負のとき最大値が 0 になる
int max = 0;

// ✅ 正解: 配列の最初の要素で初期化
int max = a[0];
```

### 間違い4: 2次元配列の行と列を逆にする

```c
int m[3][4];   // 3行、4列

// ❌ 間違い: 行と列を逆に考える
m[4][3] = 1;   // 存在しない添字！（範囲外）

// ✅ 正解: [行][列] の順
m[2][3] = 1;   // 2行3列（0始まりなので最後の行・列）
```

### 間違い5: 関数に配列を渡すときのサイズ忘れ

```c
// ❌ 間違い: サイズを渡さないと関数内で要素数が分からない
void print_array(int a[]) {
    // sizeof(a) はここでは使えない（ポインタのサイズになる）
}

// ✅ 正解: サイズも一緒に渡す
void print_array(int a[], int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", a[i]);
    }
}
```

## 9. 例題

### 例題1: 配列の最小値を求める

```c
#include <stdio.h>
int main() {
    int a[] = {64, 34, 25, 12, 22, 11, 90};
    int n = 7;
    int min_val = a[0];   // 最初の要素を最小値と仮定

    for (int i = 1; i < n; i++) {   // i=1から始める（a[0]は比較不要）
        if (a[i] < min_val) {
            min_val = a[i];
        }
    }
    printf("最小値: %d\n", min_val);   // 11
    return 0;
}
```

**トレース**:

| i | a[i] | a[i]<min_val | min_val（後） |
|---|---|---|---|
| 初期 | — | — | 64 |
| 1 | 34 | 34<64 **true** | 34 |
| 2 | 25 | 25<34 **true** | 25 |
| 3 | 12 | 12<25 **true** | 12 |
| 4 | 22 | 22<12 false | 12 |
| 5 | 11 | 11<12 **true** | 11 |
| 6 | 90 | 90<11 false | 11 |

---

### 例題2: 閾値を超える要素の個数を数える

```c
#include <stdio.h>
int main() {
    int scores[] = {85, 92, 78, 90, 65, 88, 73};
    int n = 7;
    int threshold = 80;
    int count = 0;

    for (int i = 0; i < n; i++) {
        if (scores[i] >= threshold) {
            count++;
        }
    }
    printf("%d点以上: %d人\n", threshold, count);   // 80点以上: 4人
    return 0;
}
```

### 例題3: 配列の逆順（in-place）

```c
#include <stdio.h>
int main() {
    int a[] = {1, 2, 3, 4, 5};
    int n = 5;

    // 両端から中央に向かってスワップ
    for (int i = 0; i < n / 2; i++) {
        int tmp = a[i];
        a[i] = a[n - 1 - i];
        a[n - 1 - i] = tmp;
    }

    for (int i = 0; i < n; i++) {
        printf("%d ", a[i]);   // 5 4 3 2 1
    }
    printf("\n");
    return 0;
}
```

**スワップの様子**:
```
初期:    [1, 2, 3, 4, 5]
i=0: 1↔5: [5, 2, 3, 4, 1]
i=1: 2↔4: [5, 4, 3, 2, 1]
i=2: 中央なのでスキップ（n/2=2 なので i=2 は実行されない）
結果:    [5, 4, 3, 2, 1]
```

## 10. 新潟大学試験との関連

> **配列ソート（バブルソート）は毎年出題される最重要アルゴリズム。**

**バブルソートの基本コード**（試験頻出）：
```c
void bubble_sort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                int tmp = a[j];
                a[j] = a[j + 1];
                a[j + 1] = tmp;
            }
        }
    }
}
```

このコードは丸暗記レベルまで練習すること。試験では：
1. 空欄 `(A)`, `(B)`, `(C)` を埋める問題
2. ソート過程のトレースを答える問題
3. ソート後の配列を答える問題

が出題される。

**対策**: `03_drills/L5_arrays.md` の H1（バブルソートトレース）を必ず解くこと。
